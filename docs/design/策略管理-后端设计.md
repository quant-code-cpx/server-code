## 一、模块概览

### 核心定位

| 实体 | 定位 | 生命周期 | 可变性 |
|------|------|---------|--------|
| **StrategyDraft** | 编辑中的临时配置快照 | 短期，提交或丢弃 | 随时可改，无版本 |
| **Strategy**（本模块）| 用户确认保存的可复用策略模板 | 长期 | 可更新，带版本号 |
| **BacktestRun** | 一次回测的执行记录 | 永久不可变 | 创建后只读 |

## 二、Prisma Model

```prisma
model Strategy {
  id               String   @id @default(cuid())
  userId           Int      @map("user_id")
  name             String   @db.VarChar(100)
  description      String?  @db.VarChar(500)
  strategyType     String   @map("strategy_type") @db.VarChar(50)
  strategyConfig   Json     @map("strategy_config") @db.JsonB
  backtestDefaults Json?    @map("backtest_defaults") @db.JsonB
  tags             String[] @default([])
  version          Int      @default(1)
  isPublic         Boolean  @default(false) @map("is_public")
  createdAt        DateTime @default(now()) @map("created_at")
  updatedAt        DateTime @updatedAt @map("updated_at")

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@map("strategies")
  @@unique([userId, name])
  @@index([userId, updatedAt(sort: Desc)])
  @@index([strategyType])
  @@index([isPublic])
}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| `strategyType` | 对应 `BacktestStrategyType`（MA_CROSS_SINGLE / SCREENING_ROTATION / FACTOR_RANKING / CUSTOM_POOL_REBALANCE / FACTOR_SCREENING_ROTATION） |
| `strategyConfig` | 策略特定参数（与 `BacktestRun.strategyConfig` 同构），入库前由 `StrategySchemaValidatorService` 校验 |
| `backtestDefaults` | 回测默认参数（startDate / endDate / initialCapital / benchmarkTsCode 等），发起回测时可被请求参数覆盖 |
| `tags` | 用户自定义标签（最多 10 个） |
| `version` | 每次更新 `strategyConfig` 时自增，方便追溯 |
| `isPublic` | 预留策略市场能力（当前默认 false） |

### backtestDefaults 结构（可选字段）

```typescript
interface BacktestDefaults {
  benchmarkTsCode?: string      // 默认基准（如 '000300.SH'）
  universe?: string             // 股票池（ALL_A / HS300 / 等）
  customUniverse?: string[]     // 自定义股票池列表
  initialCapital?: number       // 初始资金（元）
  rebalanceFrequency?: string   // 再平衡频率（DAILY/WEEKLY/MONTHLY/QUARTERLY）
  priceMode?: string            // 成交价格（NEXT_OPEN/NEXT_CLOSE）
  commissionRate?: number       // 手续费率（如 0.0003）
  stampDutyRate?: number        // 印花税率（如 0.0005）
  slippageBps?: number          // 滑点（bps）
}
```

## 三、API 接口

所有接口均使用 `@Post`，权限为 `JwtAuthGuard`。

| 路径 | 说明 |
|------|------|
| `POST /strategies` | 创建策略 |
| `POST /strategies/list` | 查询策略列表（分页、按类型/标签/关键词过滤） |
| `POST /strategies/detail` | 查询策略详情 |
| `POST /strategies/update` | 更新策略（修改 config 时版本号自增） |
| `POST /strategies/delete` | 删除策略 |
| `POST /strategies/clone` | 克隆策略（支持克隆公开策略） |
| `POST /strategies/run` | 基于策略发起回测 |
| `POST /strategies/schemas` | 获取所有策略类型的 JSON Schema（供前端表单渲染） |

### 业务约束

- 每用户最多 50 个策略（硬编码常量 `MAX_STRATEGIES_PER_USER`）
- 每策略最多 10 个标签
- `(userId, name)` 唯一约束（数据库层保证）

## 四、策略参数 JSON Schema 校验

### 架构

`StrategySchemaValidatorService` 封装两种能力：

1. **JSON Schema 定义**（静态，供 `POST /strategies/schemas` 返回，用于前端动态表单渲染）
2. **运行时参数校验**（委托给 `BacktestStrategyRegistryService.validateStrategyConfig`，已有完整的逻辑校验）

### 触发时机

| 场景 | 触发方式 |
|------|---------|
| `POST /strategies`（创建策略） | `StrategyService.create()` 调用 `schemaValidator.validate()` |
| `POST /strategies/update`（更新策略，含 strategyConfig）| `StrategyService.update()` 调用 `schemaValidator.validate()` |
| `POST /strategies/run`（基于策略发起回测） | `BacktestRunService.createRun()` 内部再次校验 |
| `POST /strategy-draft/submit`（草稿提交）| `StrategyDraftService.submitDraft()` 提前校验 |

### 校验失败响应（code 4001）

```json
{
  "code": 4001,
  "message": "strategyConfig.shortWindow 必须小于 longWindow",
  "data": null
}
```

## 五、模块依赖关系

```
StrategyModule
├── imports: BacktestModule
│   ├── uses: BacktestRunService (发起回测)
│   └── uses: BacktestStrategyRegistryService (参数校验)
├── providers: StrategyService, StrategySchemaValidatorService
└── exports: StrategySchemaValidatorService

StrategyDraftModule
├── imports: BacktestModule, StrategyModule
└── uses: StrategySchemaValidatorService (草稿提交前校验)
```

BacktestModule 不依赖 StrategyModule，无循环依赖。

## 六、POST /strategies/run 参数合并逻辑

`RunStrategyDto` 字段覆盖优先级（从高到低）：

1. 请求体中的显式参数（`dto.benchmarkTsCode` 等）
2. `Strategy.backtestDefaults` 中存储的默认值
3. 系统硬编码默认值

`startDate`、`endDate`、`initialCapital` 为必传字段（代表本次回测的时间窗口和资金配置），不从 `backtestDefaults` 推断。
