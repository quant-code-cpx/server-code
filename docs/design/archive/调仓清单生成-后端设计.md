# 调仓清单生成 — 后端设计（OPT-1.2）

> **对应优化项**：OPT-1.2（调仓清单生成）
> **目标**：接受目标权重配置，与当前组合持仓对比，生成考虑最小交易单位（整手）、停牌跳过、交易成本估算的完整调仓清单。**纯计算端点，不写 DB。**
> **日期**：2026-04-11

---

## 一、背景与动机

OPT-1.1 实现了"回测末日持仓 → 组合持仓"的**执行**路径（直接写库）。OPT-1.2 则提供**规划预览**：

- 用户调整目标权重后，先查看"需要买入/卖出什么、各多少手、花多少成本"，**确认后再执行**。
- OPT-1.1 的 `apply-backtest` 可在执行前调用 `rebalance-plan` 对比预览，两端点互补。
- 输入来源灵活：通过 API 直接传入目标权重列表，可对接回测末日持仓、因子筛选结果或手动配置。

---

## 二、用户操作流程

```
用户构建目标组合权重
  ├─ 来源 A: 回测末日持仓权重（从 BacktestPositionSnapshot.weight 提取）
  ├─ 来源 B: 因子筛选结果（等权 / 因子值加权）
  └─ 来源 C: 手动指定（"平安银行 20%, 招行 15%, ..."）
      │
      ▼
POST /portfolio/rebalance-plan
  → 返回调仓清单（每只股票：需买/卖/调整/保持/跳过 + 成本估算）
      │
      ▼（确认后）
POST /portfolio/apply-backtest  ─── 或手工操作
```

## 三、接口设计

### 3.1 `POST /portfolio/rebalance-plan`

纯计算，不修改任何持仓数据。

#### 请求 DTO

```typescript
export class TargetItemDto {
  @IsString()
  tsCode: string // 股票代码

  @IsNumber()
  @Min(0)
  @Max(1)
  targetWeight: number // 目标权重（0 ~ 1，相对于 totalValue）
}

export class RebalancePlanDto {
  @IsString()
  portfolioId: string // 基准组合 ID

  @IsArray()
  @ValidateNested({ each: true })
  @Type(() => TargetItemDto)
  targets: TargetItemDto[] // 目标权重列表

  @IsOptional()
  @IsEnum(OmitAction)
  omitUnspecified?: OmitAction // 未在 targets 中出现的持仓怎么处理，默认 SELL

  @IsOptional()
  @IsNumber()
  @Min(0)
  totalValue?: number // 组合总市值（元）；不传则自动用最新价估算持仓市值

  @IsOptional()
  @IsNumber()
  @Min(0)
  commissionRate?: number // 佣金率，默认 0.00025（万分之二点五）

  @IsOptional()
  @IsNumber()
  @Min(0)
  stampDutyRate?: number // 印花税率（卖出），默认 0.001（千分之一）

  @IsOptional()
  @IsNumber()
  @Min(0)
  minCommission?: number // 每笔最低佣金（元），默认 5 元
}

export enum OmitAction {
  SELL = 'SELL', // 未指定持仓全部卖出（默认）
  HOLD = 'HOLD', // 未指定持仓保持不动
}
```

**校验规则**：

- `targets` 中所有 `targetWeight` 之和不能超过 1.0（允许 ≤ 0.001 浮点误差）
- `targets` 中不能出现重复 `tsCode`

#### 响应 DTO

```typescript
export class RebalancePlanItemDto {
  tsCode: string
  stockName: string

  // ── 当前状态 ────────────────────────────────────────────────────────────
  currentShares: number // 当前持仓股数
  currentPrice: number | null // 最新收盘价（元）
  currentMarketValue: number | null // 当前市值（元）
  currentWeight: number | null // 当前持仓权重（相对于 totalValue）

  // ── 目标状态 ────────────────────────────────────────────────────────────
  targetWeight: number // 输入的目标权重
  targetShares: number // 计算并整手处理后的目标股数
  targetMarketValue: number | null // 目标市值（元）

  // ── 差异与动作 ──────────────────────────────────────────────────────────
  action: 'BUY' | 'SELL' | 'ADJUST' | 'HOLD' | 'SKIP'
  skipReason?: 'SUSPENDED' | 'NO_PRICE' | 'LOT_SIZE'
  // SUSPENDED = 当日停牌
  // NO_PRICE  = 最新价格数据缺失
  // LOT_SIZE  = 目标权重过小，按当前价格买不到 1 手（100 股）

  deltaShares: number // 需要操作的股数（正=买入 负=卖出 0=不变）
  deltaAmount: number | null // 对应金额（元，始终为正，方向由 action 区分）
  estimatedTradingCost: number // 预估交易成本（元）
}

export class RebalancePlanSummaryDto {
  totalBuyAmount: number // 买入总金额（元）
  totalSellProceeds: number // 卖出总金额（元，扣除税费前）
  totalTradingCost: number // 所有买卖预估成本之和（元）
  buyCount: number
  sellCount: number
  adjustCount: number
  holdCount: number
  skipCount: number
  cashBefore: number // 预估调仓前现金余额（元）
  cashAfter: number // 预估调仓后现金余额（元）
  isFeasible: boolean // 调仓后现金是否 >= 0
}

export class RebalancePlanResponseDto {
  portfolioId: string
  portfolioName: string
  refDate: string // 参考定价日期（YYYY-MM-DD）
  totalValue: number // 使用的组合总市值（元）
  items: RebalancePlanItemDto[]
  summary: RebalancePlanSummaryDto
}
```

## 四、核心逻辑

### 4.1 主流程

```
rebalancePlan(dto, userId)
  │
  ├─ 1. 权限校验
  │     ├─ portfolio.findUnique(id)
  │     ├─ 不存在 → NotFoundException
  │     └─ userId 不匹配 → ForbiddenException
  │
  ├─ 2. 输入校验
  │     ├─ sum(targets.targetWeight) > 1.001 → BadRequestException
  │     └─ targets 中 tsCode 重复 → BadRequestException
  │
  ├─ 3. 读取当前持仓
  │     └─ portfolioHolding.findMany({ portfolioId })
  │
  ├─ 4. 查询最新交易日 & 价格
  │     ├─ tradeCal.findFirst(isOpen=1, orderBy desc) → latestDate
  │     └─ daily.findMany(tsCode in allCodes, tradeDate=latestDate)
  │         → priceMap: Map<tsCode, close>
  │
  ├─ 5. 查询停牌状态
  │     └─ suspendD.findMany(tradeDate=latestDateStr)
  │         → suspendedSet: Set<tsCode>
  │         注：SuspendD.tradeDate 是 String(YYYYMMDD)，需格式转换
  │
  ├─ 6. 补全股票名称
  │     └─ stockBasic.findMany(tsCode in allCodes) → nameMap
  │
  ├─ 7. 确定 totalValue
  │     ├─ dto.totalValue 有值 → 直接使用
  │     └─ 否则 → Σ(current.quantity × price) for all holdings with price
  │              （无价格的持仓用成本价 avgCost 兜底）
  │
  ├─ 8. 整手计算函数
  │     roundToLot(totalValue, weight, price):
  │       rawShares = floor(totalValue × weight / price)
  │       return floor(rawShares / 100) × 100
  │
  ├─ 9. 生成调仓计划 (items)
  │     ├─ 遍历 targets 中每个 targetItem:
  │     │   ├─ 停牌 → SKIP(SUSPENDED)
  │     │   ├─ price = null → SKIP(NO_PRICE)
  │     │   ├─ roundToLot(...) = 0 && targetWeight > 0 → SKIP(LOT_SIZE)
  │     │   └─ 与当前持仓 diff:
  │     │       ├─ 现有 = 0 → BUY
  │     │       ├─ 目标 = 0 → SELL
  │     │       ├─ 目标股数 = 现有股数 → HOLD
  │     │       └─ 其他 → ADJUST
  │     │
  │     └─ 遍历当前持仓中不在 targets 里的股票:
  │         ├─ omitUnspecified = SELL → action=SELL (全部卖出，无整手约束)
  │         └─ omitUnspecified = HOLD → action=HOLD
  │
  ├─ 10. 估算交易成本
  │      BUY:  max(amount × commissionRate, minCommission)
  │      SELL: max(amount × commissionRate, minCommission) + amount × stampDutyRate
  │
  ├─ 11. 估算现金变动
  │      cashBefore = portfolio.initialCash - Σ(holding.quantity × holding.avgCost)
  │      totalBuyAmount = Σ(BUY/ADJUST增仓 的 deltaAmount)
  │      totalSellProceeds = Σ(SELL/ADJUST减仓 的 deltaAmount)
  │      cashAfter = cashBefore + totalSellProceeds - totalBuyAmount - totalTradingCost
  │      isFeasible = cashAfter >= 0
  │
  └─ 12. 组装并返回 RebalancePlanResponseDto
```

### 4.2 整手计算细节

```typescript
// 正常买入：向下取整到 100 倍数
function roundToLot(totalValue: number, targetWeight: number, price: number): number {
  if (price <= 0) return 0
  const rawShares = Math.floor((totalValue * targetWeight) / price)
  return Math.floor(rawShares / 100) * 100
}

// 特殊情况：持仓清仓（targetWeight = 0）
// → 直接使用 currentShares（不受 100 股整手限制，允许零股全清）
```

### 4.3 停牌日期格式转换

`SuspendD.tradeDate` 存储为 `String` 类型（`'YYYYMMDD'` 格式），而 `tradeCal.calDate` 返回的是 `Date` 对象，需在查询前转换：

```typescript
// Date → 'YYYYMMDD' string
function toTushareDateStr(date: Date): string {
  return date.toISOString().slice(0, 10).replace(/-/g, '')
}
```

### 4.4 交易成本计算

| 方向 | 公式                                                                   |
| ---- | ---------------------------------------------------------------------- |
| 买入 | `max(amount × commissionRate, minCommission)`                          |
| 卖出 | `max(amount × commissionRate, minCommission) + amount × stampDutyRate` |

**默认参数**：
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `commissionRate` | `0.00025` | 万分之二点五 |
| `stampDutyRate` | `0.001` | 千分之一（仅卖出） |
| `minCommission` | `5` | 每笔最低 5 元 |

### 4.5 SKIP 场景

| `skipReason` | 触发条件                                    | 说明                               |
| ------------ | ------------------------------------------- | ---------------------------------- |
| `SUSPENDED`  | `suspendedSet.has(tsCode)`                  | 当日停牌，无法成交                 |
| `NO_PRICE`   | `priceMap.get(tsCode) == null`              | 最新日线价格数据缺失               |
| `LOT_SIZE`   | `roundToLot(...) === 0 && targetWeight > 0` | 目标权重过小，买不到 1 手（100股） |

SKIP 的标的：`deltaShares = 0`，`deltaAmount = null`，`estimatedTradingCost = 0`。

## 十、后续扩展点

- **涨跌停检测**：`DailyBasic.limitStatus` 或 `StkLimit` 表，跳过当日涨跌停股票（无法成交），可追加 `skipReason: 'LIMIT_UP' | 'LIMIT_DOWN'`
- **OPT-3.1 盘后信号引擎**：信号生成后可直接调用 `rebalance-plan` 生成执行清单，再由用户确认后走 `apply-backtest`
- **多账户资金适配**：当前 `cashBefore` 基于 `initialCash - 成本`，后续可接交易日志（OPT-4.4）得到真实现金余额
- **T+1 限制**：当日买入的股票次日才能卖出，可作为 `skipReason: 'T1_RESTRICTION'` 追加（需关联交易日志）
