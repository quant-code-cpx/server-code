## 2. 核心业务规则

| 规则编号 | 规则 | 来源 | 测试影响 |
| -------- | ---- | ---- | -------- |
| R-US01 | account 必填，最大 64 字符 | CreateUserDto @IsString @IsNotEmpty @MaxLength(64) | 缺失/空/超长应 400 |
| R-US02 | nickname 必填，最大 64 字符 | CreateUserDto @IsString @IsNotEmpty @MaxLength(64) | 缺失/空/超长应 400 |
| R-US03 | password 必填，最少 8 位 | CreateUserDto @IsString @IsNotEmpty @MinLength(8) | 缺失/空/不足 8 位应 400 |
| R-US04 | role 必须是 UserRole 枚举 | CreateUserDto @IsEnum(UserRole) | 无效应 400 |
| R-US05 | backtestQuota/watchlistLimit 最小 -1 | CreateUserDto @Min(-1) | 小于 -1 应 400 |
| R-US06 | 新密码最少 8 位 | ChangePasswordDto @MinLength(8) | 不足 8 位应 400 |
| R-US07 | id 必须是正整数 | UserIdDto @IsInt @IsPositive | 非正整数应 400 |
| R-US08 | status 必须是 ACTIVE/DEACTIVATED | UpdateUserStatusDto @IsEnum | 无效应 400 |
| R-US09 | pageSize 1~100 | UserListQueryDto @Min(1) @Max(100) | 超范围应 400 |
| R-US10 | page 1~1000 | UserListQueryDto @Min(1) @Max(1000) | 超范围应 400 |
| R-US11 | email 格式校验 | UpdateProfileDto @IsEmail | 格式错误应 400 |
| R-US12 | keyword 必填，最大 64 字符 | UserSearchQueryDto @IsString @MaxLength(64) | 缺失/超长应 400 |
| R-US13 | reset-password newPassword 至少 8 位 | ResetPasswordDto @MinLength(8) | 不足 8 位应 400 |
| R-US14 | update-role 不可设为 SUPER_ADMIN | UserService 业务逻辑 | 设为 SUPER_ADMIN 应抛业务异常 |
| R-US15 | 管理员端点需 ADMIN 以上角色 | @Roles(UserRole.ADMIN) + RolesGuard | USER 角色应 403 |
| R-US16 | update-role/restore 需 SUPER_ADMIN | @Roles(UserRole.SUPER_ADMIN) + RolesGuard | ADMIN 应 403 |
| R-US17 | 只能操作低于自身角色的用户 | UserService.assertHigherRole | 操作同级/高级应抛业务异常 |
| R-US18 | 不能操作 SUPER_ADMIN 用户 | UserService.assertNotSuperAdmin | 操作 SUPER_ADMIN 应抛业务异常 |
