## 一、功能总览

用户可基于已有的回测结果、持仓组合、个股分析数据，生成结构化的量化报告，支持 JSON / HTML / PDF 三种输出格式。

| 接口                  | 路径                          | 功能说明                                         | 是否需新建表 |
| --------------------- | ----------------------------- | ------------------------------------------------ | ------------ |
| 生成回测报告          | `POST /report/backtest`       | 基于 BacktestRun，生成含净值曲线、交易明细的报告 | 是（1 张）   |
| 生成个股研报          | `POST /report/stock`          | 基于个股行情 + 技术指标 + 财务数据               | 否           |
| 生成组合分析报告      | `POST /report/portfolio`      | 基于 Portfolio 持仓快照 + 风险因子               | 否           |
| 查询报告列表          | `GET /report`                 | 查询用户历史报告                                 | 否           |
| 下载报告              | `GET /report/:id/download`    | 下载 PDF / HTML 文件                             | 否           |
| 获取报告 JSON 数据    | `GET /report/:id/data`        | 获取报告结构化 JSON 数据（前端自渲染用）         | 否           |

## 三、报告类型与数据源

### 3.1 回测报告（Backtest Report）

| 章节         | 数据源                                | 说明                                       |
| ------------ | ------------------------------------- | ------------------------------------------ |
| 策略概要     | `BacktestRun`                         | 策略名、参数、回测区间、基准               |
| 绩效指标     | `BacktestRun.metrics`                 | 年化收益、最大回撤、夏普、胜率等           |
| 净值曲线     | `BacktestDailyNav`                    | 策略净值 vs 基准净值时间序列               |
| 回撤分析     | `BacktestDailyNav.drawdown`           | 回撤曲线 + 最大回撤区间标注               |
| 月度收益     | 由 `BacktestDailyNav` 聚合           | 月度收益热力图                             |
| 交易明细     | `BacktestTrade`                       | 买卖信号 + 成本                            |
| 持仓快照     | `BacktestPositionSnapshot`            | 末期持仓分布                               |
| 调仓日志     | `BacktestRebalanceLog`                | 调仓执行记录                               |

### 3.2 个股研报（Stock Research Report）

| 章节           | 数据源                       | 说明                                   |
| -------------- | ---------------------------- | -------------------------------------- |
| 公司概况       | `StockBasic` + `StockCompany`| 行业、上市日期、主营业务               |
| 行情走势       | `Daily` + `AdjFactor`        | 近 1 年 / 6 月 / 3 月 K 线 + 均线      |
| 技术指标       | `computeAllIndicators()`     | MACD / RSI / KDJ / BOLL 最新值与趋势  |
| 财务概况       | `FinaIndicator`              | ROE / 净利润率 / 营收增长率近 4 季     |
| 利润表摘要     | `Income`                     | 近 3 年营收、净利润趋势                |
| 资产负债摘要   | `BalanceSheet`               | 总资产、净资产、资产负债率             |
| 股东信息       | `Top10Holders`               | 前十大股东及变动                       |
| 分红送配       | `Dividend`                   | 近年分红记录                           |

### 3.3 组合分析报告（Portfolio Report）

| 章节           | 数据源                          | 说明                                 |
| -------------- | ------------------------------- | ------------------------------------ |
| 组合概要       | `Portfolio`                     | 组合名、创建日期、总市值             |
| 持仓明细       | `PortfolioPosition`             | 个股 / 权重 / 盈亏                   |
| 行业分布       | 持仓 + `StockBasic.industry`    | 行业权重饼图数据                     |
| 风险因子暴露   | `FactorSnapshot` 截面           | 市值/估值/动量/波动率等因子暴露      |
| 相关性矩阵     | 持仓日收益率                    | 持仓间相关系数矩阵                   |
| 集中度分析     | 持仓权重                        | 前 N 大持仓权重 / HHI 指数          |

```prisma
/// 报告元数据
model Report {
  id            String       @id @default(uuid()) @db.Uuid
  userId        String       @map("user_id") @db.Uuid
  type          ReportType   @map("report_type")
  title         String       @db.VarChar(200)
  /// 生成报告时使用的参数（JSON）
  params        Json         @db.JsonB
  /// 报告结构化数据（JSON），前端可直接渲染
  data          Json?        @db.JsonB
  /// 报告文件存储路径（HTML/PDF）
  filePath      String?      @map("file_path") @db.VarChar(500)
  format        ReportFormat @default(JSON)
  status        ReportStatus @default(PENDING)
  errorMessage  String?      @map("error_message") @db.Text
  fileSize      Int?         @map("file_size")
  createdAt     DateTime     @default(now()) @map("created_at") @db.Timestamptz()
  completedAt   DateTime?    @map("completed_at") @db.Timestamptz()

  user          User         @relation(fields: [userId], references: [id])

  @@index([userId, createdAt(sort: Desc)])
  @@index([type])
  @@map("reports")
}

enum ReportType {
  BACKTEST
  STOCK
  PORTFOLIO

  @@map("report_type")
}

enum ReportFormat {
  JSON
  HTML
  PDF

  @@map("report_format")
}

enum ReportStatus {
  PENDING
  GENERATING
  COMPLETED
  FAILED

  @@map("report_status")
}
```

```typescript
@Injectable()
export class ReportService {
  private readonly logger = new Logger(ReportService.name)

  constructor(
    private readonly prisma: PrismaService,
    private readonly reportDataCollector: ReportDataCollectorService,
    private readonly reportRenderer: ReportRendererService,
  ) {}

  /** 生成回测报告 */
  async createBacktestReport(userId: string, dto: CreateBacktestReportDto): Promise<ReportSummaryDto> {
    // 1. 验证回测 runId 存在并属于该用户
    const run = await this.prisma.backtestRun.findFirstOrThrow({
      where: { id: dto.runId, userId },
    })

    // 2. 创建报告记录（PENDING）
    const report = await this.prisma.report.create({
      data: {
        userId,
        type: 'BACKTEST',
        title: dto.title ?? `回测报告 - ${run.name ?? run.id}`,
        params: { runId: dto.runId },
        format: dto.format ?? 'JSON',
        status: 'GENERATING',
      },
    })

    try {
      // 3. 收集报告数据
      const reportData = await this.reportDataCollector.collectBacktestData(dto.runId)

      // 4. 渲染（如需 HTML/PDF）
      let filePath: string | null = null
      let fileSize: number | null = null
      if (dto.format === 'HTML' || dto.format === 'PDF') {
        const rendered = await this.reportRenderer.render('backtest', reportData, dto.format as 'HTML' | 'PDF')
        filePath = rendered.filePath
        fileSize = rendered.fileSize
      }

      // 5. 更新报告记录
      await this.prisma.report.update({
        where: { id: report.id },
        data: {
          data: reportData as any,
          filePath,
          fileSize,
          status: 'COMPLETED',
          completedAt: new Date(),
        },
      })

      return this.toSummary({ ...report, status: 'COMPLETED', filePath, fileSize, completedAt: new Date() })
    } catch (error) {
      await this.prisma.report.update({
        where: { id: report.id },
        data: { status: 'FAILED', errorMessage: error.message },
      })
      throw error
    }
  }

  /** 生成个股研报 */
  async createStockReport(userId: string, dto: CreateStockReportDto): Promise<ReportSummaryDto> {
    // 类似流程：验证 → 创建记录 → 收集数据 → 渲染 → 更新状态
  }

  /** 生成组合报告 */
  async createPortfolioReport(userId: string, dto: CreatePortfolioReportDto): Promise<ReportSummaryDto> {
    // 类似流程
  }

  /** 查询报告列表 */
  async queryReports(userId: string, dto: QueryReportsDto) {
    const where: any = { userId }
    if (dto.type) where.type = dto.type
    const [items, total] = await Promise.all([
      this.prisma.report.findMany({
        where,
        orderBy: { createdAt: 'desc' },
        skip: ((dto.page ?? 1) - 1) * (dto.pageSize ?? 20),
        take: dto.pageSize ?? 20,
        select: { id: true, type: true, title: true, format: true, status: true, fileSize: true, createdAt: true, completedAt: true },
      }),
      this.prisma.report.count({ where }),
    ])
    return { items, total, page: dto.page ?? 1, pageSize: dto.pageSize ?? 20 }
  }

  /** 获取报告 JSON 数据 */
  async getReportData(userId: string, reportId: string) {
    const report = await this.prisma.report.findFirstOrThrow({
      where: { id: reportId, userId },
    })
    return report.data
  }

  /** 下载报告文件 */
  async getReportFile(userId: string, reportId: string): Promise<{ filePath: string; format: string }> {
    const report = await this.prisma.report.findFirstOrThrow({
      where: { id: reportId, userId, status: 'COMPLETED' },
    })
    if (!report.filePath) throw new BusinessException('该报告无可下载文件')
    return { filePath: report.filePath, format: report.format }
  }

  private toSummary(report: any): ReportSummaryDto {
    return {
      id: report.id,
      type: report.type,
      title: report.title,
      format: report.format,
      status: report.status,
      fileSize: report.fileSize,
      createdAt: report.createdAt,
      completedAt: report.completedAt,
    }
  }
}
```

```typescript
  constructor(
    private readonly prisma: PrismaService,
    // 复用已有服务而非重新实现查询逻辑
  ) {}

  /** 采集回测报告数据 */
  async collectBacktestData(runId: string): Promise<BacktestReportDataDto> {
    const [run, navs, trades, positions] = await Promise.all([
      this.prisma.backtestRun.findUniqueOrThrow({ where: { id: runId } }),
      this.prisma.backtestDailyNav.findMany({
        where: { runId },
        orderBy: { tradeDate: 'asc' },
      }),
      this.prisma.backtestTrade.findMany({
        where: { runId },
        orderBy: { tradeDate: 'asc' },
      }),
      this.prisma.backtestPositionSnapshot.findMany({
        where: { runId },
        orderBy: { tradeDate: 'desc' },
        // 只取最后一天持仓
      }),
    ])

    // 聚合月度收益
    const monthlyReturns = this.aggregateMonthlyReturns(navs)

    return {
      strategy: {
        name: run.name ?? run.id,
        params: run.config as Record<string, unknown>,
        startDate: run.startDate,
        endDate: run.endDate,
        benchmark: run.benchmark ?? '000300.SH',
        initialCapital: run.initialCapital?.toNumber() ?? 1_000_000,

      metrics: {
        totalReturn: run.totalReturn?.toNumber() ?? 0,
        annualizedReturn: run.annualizedReturn?.toNumber() ?? 0,

        sharpeRatio: run.sharpeRatio?.toNumber() ?? 0,
```

```typescript
@ApiTags('Report - 量化报告')

  @ApiOperation({ summary: '获取报告 JSON 数据' })
  getReportData(@CurrentUser() user, @Param('id', ParseUUIDPipe) id: string) {
    return this.reportService.getReportData(user.id, id)
  }

  @Get(':id/download')
  @ApiOperation({ summary: '下载报告文件（HTML / PDF）' })
  async downloadReport(
    @CurrentUser() user,
    @Param('id', ParseUUIDPipe) id: string,
    @Res() res: Response,
  ) {
    const { filePath, format } = await this.reportService.getReportFile(user.id, id)
    const contentType = format === 'PDF' ? 'application/pdf' : 'text/html'
    res.setHeader('Content-Type', contentType)
    res.setHeader('Content-Disposition', `attachment; filename="report.${format.toLowerCase()}"`)
    const stream = createReadStream(filePath)
    stream.pipe(res)
  }
}
```

### 5.3 渲染服务

**文件**：`src/apps/report/services/report-renderer.service.ts`

```typescript
@Injectable()
export class ReportRendererService {
  private readonly compiledTemplates = new Map<string, HandlebarsTemplateDelegate>()

  constructor() {
    this.registerHelpers()
    this.precompileTemplates()
  }

  /** 渲染报告为 HTML 或 PDF */
  async render(
    templateName: string,
    data: Record<string, unknown>,
    format: 'HTML' | 'PDF',
  ): Promise<{ filePath: string; fileSize: number }> {
    // 1. 渲染 HTML
    const template = this.compiledTemplates.get(templateName)
    if (!template) throw new Error(`Template "${templateName}" not found`)
    const html = template(data)

    // 2. 保存 HTML 文件
    const outputDir = join(process.cwd(), 'storage', 'reports')
    await ensureDir(outputDir)
    const baseName = `${templateName}-${Date.now()}`

    if (format === 'HTML') {
      const htmlPath = join(outputDir, `${baseName}.html`)
      await writeFile(htmlPath, html, 'utf-8')
      const stat = await fsStat(htmlPath)
      return { filePath: htmlPath, fileSize: stat.size }
    }

    // 3. PDF：使用 Puppeteer 将 HTML 转为 PDF
    const pdfPath = join(outputDir, `${baseName}.pdf`)
    const browser = await puppeteer.launch({
      headless: true,
      args: ['--no-sandbox', '--disable-setuid-sandbox'],
    })
    try {
      const page = await browser.newPage()
      await page.setContent(html, { waitUntil: 'networkidle0' })
      await page.pdf({
        path: pdfPath,
        format: 'A4',
        margin: { top: '20mm', bottom: '20mm', left: '15mm', right: '15mm' },
        printBackground: true,
      })
    } finally {
      await browser.close()
    }
    const stat = await fsStat(pdfPath)
    return { filePath: pdfPath, fileSize: stat.size }
  }

  private registerHelpers() {
    Handlebars.registerHelper('formatPercent', (val: number) =>
      val != null ? `${(val * 100).toFixed(2)}%` : '-',
    )
    Handlebars.registerHelper('formatNumber', (val: number, decimals: number) =>
      val != null ? val.toFixed(decimals) : '-',
    )
    Handlebars.registerHelper('gt', (a: number, b: number) => a > b)
  }

  private precompileTemplates() {
    const templateDir = join(__dirname, '..', 'templates')
    for (const name of ['backtest', 'stock', 'portfolio']) {
      const tplPath = join(templateDir, `${name}.hbs`)
      if (existsSync(tplPath)) {
        const source = readFileSync(tplPath, 'utf-8')
        this.compiledTemplates.set(name, Handlebars.compile(source))
      }
    }
  }
}
```

## 六、PDF 生成方案

### 6.1 技术选型

| 方案                | 优点                            | 缺点                           | 推荐度 |
| ------------------- | ------------------------------- | ------------------------------ | ------ |
| **Puppeteer**       | HTML→PDF 最高保真度、支持图表   | 需要 Chromium（~200MB）          | ⭐⭐⭐ |
| `pdfkit`            | 纯 Node.js、体积小             | 排版需手写坐标、不支持 HTML    | ⭐     |
| `@pdfme/generator`  | 模板可视化编辑                  | 生态小、复杂报告支持弱         | ⭐     |

**推荐 Puppeteer**：

1. 报告可先预览 HTML，再生成 PDF，一套模板两用
2. Docker 环境可用 `puppeteer-core` + 预装 Chromium，镜像仅增加 ~100MB
3. 图表数据以 JSON 嵌入 HTML，前端渲染后由 Puppeteer 截取静态图

### 6.2 安装依赖

```bash
pnpm add handlebars puppeteer

pnpm add handlebars puppeteer-core
```
