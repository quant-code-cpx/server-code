## 3. 业务理解

### 3.1 业务场景

- 历史研究必须只看到 `availableAt` 当时已实际公告的报表和指标，不能穿越到未来修订版。
- 财报流量项需同时说明累计值与单季值；资产负债表是期末时点值，不做单季相减。
- 指标只能通过服务端 allowlist 读取，返回 canonical key、上游字段和单位。
- 个股资金流保留 Tushare `net_mf_amount` 原值；分单买卖只作结构化明细，不反推官方净流入。

### 3.2 核心业务规则

| 规则编号 | 规则                                                                                                      | 来源                        | 测试影响                                   |
| -------- | --------------------------------------------------------------------------------------------------------- | --------------------------- | ------------------------------------------ |
| R-001    | 实际可得日取 `fAnnDate ?? annDate`；历史查询只允许可得日不晚于 `availableAt` 的版本                       | Agent 数据边界              | 同报告期前后两个 as-of 结果不同            |
| R-002    | 同报告期优先 `updateFlag=1`，再按实际可得日、同步时间和稳定 ID 选版；返回 revision warning                | Tushare 更新标识 + 实库重复 | 修订不会随机漂移                           |
| R-003    | `QUARTERLY` 包含 03-31/06-30/09-30/12-31；`ANNUAL` 只含 12-31                                             | Tool period 契约            | 过滤和 limit 结果稳定                      |
| R-004    | 利润表、现金流 Q1 单季值等于累计值；Q2/Q3/Q4 = 当期累计 - 同年上一季度累计                                | 累计财报会计口径            | 用 100、260、450、700 手算 100/160/190/250 |
| R-005    | 缺上一季度或任一操作数为 null 时，单季派生值为 null 并告警；null 不转 0                                   | 数据完整性规则              | 防止虚构单季利润/现金流                    |
| R-006    | 资产负债表字段标记 `POINT_IN_TIME`，不做累计或单季派生                                                    | 资产负债表时点口径          | 防止资产余额被相减                         |
| R-007    | `FinaIndicator` 仅一行/报告期，历史修订版本不可恢复；历史查询返回 PIT revision warning                    | Prisma PK 与实库结构        | 不宣称完整 point-in-time 修订历史          |
| R-008    | 指标最多 30 个 allowlist key；金额、每股、百分数、比率单位分别返回                                        | Batch 008 + Prisma 注释     | 未知字段在查询前拒绝                       |
| R-009    | `net_mf_amount` 为 L2 主动买卖净值，不能用各分单总和相减替代；金额万元、量手                              | Tushare moneyflow 官方说明  | golden fixture 故意让两者不相等            |
| R-010    | 资金流日期条件下推 DB，最大 250 日；结果按交易日升序，超量返回 `truncated=true`                           | Tool Schema                 | 不用当前 `days` 近似日期范围               |
| R-011    | 三 Tool 均 READ/idempotent；不触发同步、补数、业务写入或私有缓存                                          | Agent 只读边界              | 重复/并发调用无状态变化                    |
| R-012    | 空记录返回 typed `DATA_NOT_FOUND`；数据质量异常与 DB 失败分别映射 `DATA_QUALITY_FAILED`/`UPSTREAM_FAILED` | Tool 错误契约               | 不用空成功掩盖异常                         |

### 3.3 状态变化与不变量

| 类型           | 说明                                                                              |
| -------------- | --------------------------------------------------------------------------------- |
| 数据写入       | 无                                                                                |
| 缓存/队列/通知 | 无；本批不缓存财务或资金流                                                        |
| 权限边界       | 只读公共数据；模型输入不能包含 `userId`、表名、where、select、orderBy 或任意字段  |
| 时间规则       | Tool 使用 ISO 日期；DB 使用报告期/公告日/交易日；时区固定 `Asia/Shanghai`         |
| 数据不变量     | null 保持 null；单位完整；升序稳定；输出期数/指标/天数受硬上限；provenance 可追溯 |

## 4. Tool 契约

| Tool                       | 必填输入                                  | 最大结果               | 主要响应                                                                   |
| -------------------------- | ----------------------------------------- | ---------------------- | -------------------------------------------------------------------------- |
| `get_financial_statements` | tsCode、statementTypes、periodType、limit | 12 个报告期 × 3 类报表 | reportPeriod、announcementDate、reportType、revision、固定字段值/单位/口径 |
| `get_financial_indicators` | tsCode、indicators、limit                 | 20 期 × 30 指标        | reportPeriod、announcementDate、canonical/source field、value、unit        |
| `get_stock_moneyflow`      | tsCode、startDate、endDate                | 250 日                 | tradeDate、官方净流入、净量、可选四档买卖、单位                            |

三个输入 schema 均 `additionalProperties=false`；adapter 只依赖领域 Facade，不直接注入 Prisma 或调用内部 HTTP。

## 6. 风险分析

| 风险                        | 业务影响                    | 测试策略                                  | 优先级 |
| --------------------------- | --------------------------- | ----------------------------------------- | ------ |
| 未来公告/修订穿越           | 回测和历史研究失真          | 前后两个 availableAt + 多修订 fixture     | P0     |
| 累计值误当单季值            | 盈利趋势和现金流结论错误    | 四季度手算 golden                         | P0     |
| updateFlag 重复导致随机选版 | 同输入多次结果漂移          | 打乱输入顺序，验证稳定优先级              | P0     |
| 官方净流入被分单重算        | 资金方向和金额错误          | 构造官方净额与分单差不相等 fixture        | P0     |
| null 被转 0                 | 缺失数据伪装成真实零值      | 各层 null contract                        | P0     |
| 任意指标/字段注入           | 数据泄露、慢查询或 SQL 风险 | strict schema + allowlist + Facade 零查询 | P0     |
| 大范围查询或全表扫描        | DB 延迟和 Agent 结果膨胀    | limit+1 截断、日期下推、EXPLAIN           | P1     |
