## 1. 本轮目标

- 让 Agent 可查询最新公开网页，但不能提交任意 URL。
- 让搜索摘要只用于候选排序；关键事实必须抓取正文并绑定内容 hash 与 locator。
- 验证生产抓取只允许 HTTPS 默认端口，逐跳阻断 SSRF、DNS rebinding、私网地址和危险 MIME。
- 验证网页正文始终作为不可信外部内容，疑似 Prompt Injection 必须显式标记。
- 验证默认搜索 Provider 和 Tool allowlist 仍为关闭状态，缺少真实 API key 不影响应用启动。

## 2. 实现结果

| 能力             | 实现                                        | 关键边界                                                                                    |
| ---------------- | ------------------------------------------- | ------------------------------------------------------------------------------------------- |
| `search_web`     | `WebSearchService` + provider-neutral port  | 最多 10 条；canonicalize/去重；snippet 不可作为强引用                                       |
| `fetch_web_page` | `WebFetchService` + `SafeWebFetcherService` | 只接受当前 user/run 的签名 token；最多 100,000 字符                                         |
| 搜索 Provider    | `disabled`、`fake`、`brave`                 | 默认 `disabled`；生产禁止 `fake`；Brave 只允许官方 HTTPS endpoint                           |
| URL token        | HMAC-SHA256                                 | 绑定 `sourceId/userId/runId/urlHash/expiry`；不在 token 放完整 URL                          |
| SSRF policy      | DNS 预解析 + request lookup pinning         | 拒绝 loopback、RFC1918、link-local、metadata、IPv6 ULA/mapped-private、userinfo、非默认端口 |
| 网页抽取         | 静态 HTML/纯文本抽取                        | 不执行 JavaScript；移除 script/style/form/hidden；生成 paragraph/offset locator             |
| 引用来源         | `AiSearchSource` + `CitationRepository`     | 保存 canonical URL、fetch 状态、内容 hash、提取版本、时点和 locator metadata                |
| Prompt Injection | 搜索摘要与正文特征检测                      | 输出 warning/risk flag；正文固定标记 `untrustedExternalContent=true`                        |

Tool 细分来源类型 `EXCHANGE/REGULATOR/COMPANY` 持久化时映射到现有 `AiSourceType.OFFICIAL`；Tool 输出仍保留细分类。不新增 Prisma enum 或 migration。

## 3. 配置与默认行为

新增环境变量：

- `AGENT_SEARCH_PROVIDER=disabled`
- `AGENT_SEARCH_API_KEY`
- `AGENT_SEARCH_BASE_URL=https://api.search.brave.com/res/v1/web/search`
- `AGENT_SEARCH_TIMEOUT_MS=10000`
- `AGENT_FETCH_MAX_BYTES=1000000`
- `AGENT_FETCH_MAX_REDIRECTS=3`
- `AGENT_URL_TOKEN_SECRET`
- `AGENT_URL_TOKEN_TTL_SECONDS=900`

默认 `AGENT_SEARCH_PROVIDER=disabled`，默认 `AGENT_TOOLS_ENABLED` 仍为空。只有显式配置 Provider、密钥、URL token secret 和最小 Tool allowlist 后才会开放联网能力。

HTTP 只能通过构造参数注入精确测试 fixture host；不存在环境变量或生产配置放开路径。

## 7. 安全与范围结论

- 模型不能传 URL；只能传 `search_web` 签发的短期 token。
- token 验证 user/run/urlHash；数据库来源 URL 变化会使 token 失效。
- 每一跳重新执行协议、端口、hostname、DNS 和地址策略；连接使用已校验地址，不二次自由解析。
- 不携带 Cookie、Authorization、用户浏览器登录态或内部 header。
- 只允许 HTML/XHTML/纯文本；不下载 PDF/可执行文件，不执行 JavaScript。
- 搜索 query 日志只保存 hash；fetch 日志只保存 host hash。
- 本批无公共 REST、无 migration、无新 npm 依赖、无浏览器渲染或对象存储。
