## 一、重构背景

选股器 V1 已实现基础多维度筛选（估值、行情、财务、资金流、技术指标），但项目后续接入了大量新数据源，选股器尚未利用这些数据。本次重构旨在：

1. **新增概念板块筛选** — 利用 `ths_index_boards` + `ths_index_members` 实现概念/行业板块维度筛选
2. **多行业/多地域支持** — 从单选升级为多选（数组）
3. **增加 PS(TTM) 筛选** — 利用 `stock_daily_valuation_metrics` 的 ps_ttm 字段
4. **增加布林带信号** — 价格突破上轨 / 跌破下轨 / 缩口
5. **增加均线趋势信号** — MA 多头排列（ma5 > ma10 > ma20）、MA 空头排列
6. **增加北向资金筛选** — 利用 `hk_hold_detail` 筛选北向持仓股票
7. **扩展内置预设** — 新增"北向重仓"、"技术突破"、"概念热门"等预设
8. **返回字段增强** — 新增 PS、概念列表、北向持仓比例等
9. **新增概念列表端点** — 供前端概念板块筛选器使用

## 四、新增端点

### 4.1 `POST /stock/screener/concepts` — 可用概念列表

**功能**：返回同花顺概念板块列表，供前端下拉多选。

**响应结构**：

```typescript
interface ScreenerConceptListResponse {
  concepts: Array<{
    tsCode: string   // 板块代码
    name: string     // 板块名称
    count: number    // 成分股数量
  }>
}
```

**SQL**：

```sql
SELECT ti.ts_code, ti.name, COUNT(tm.con_code) AS count
FROM ths_index_boards ti
JOIN ths_index_members tm ON tm.ts_code = ti.ts_code AND tm.is_new = 'Y'
WHERE ti.type = 'N'
GROUP BY ti.ts_code, ti.name
HAVING COUNT(tm.con_code) > 0
ORDER BY COUNT(tm.con_code) DESC
```

**缓存**：Redis key `stock:screener_concepts`, TTL 24h。

## 五、SQL 变更

### 5.1 概念板块 JOIN

当传入 `conceptCodes` 时：

```sql
INNER JOIN ths_index_members tm ON tm.con_code = sb.ts_code AND tm.is_new = 'Y'
  AND tm.ts_code = ANY(${conceptCodes})
```

> 注意使用 `INNER JOIN` 而非 LEFT JOIN，因为概念筛选要求股票必须属于指定概念。
> 当选择多个概念时使用 `ANY` 匹配，结果做 `DISTINCT` 去重。

### 5.2 多行业/多地域

```sql
-- industries 数组
WHERE sb.industry = ANY(${industries})

-- areas 数组
WHERE sb.area = ANY(${areas})
```

### 5.3 PS(TTM) 筛选

```sql
-- 已在 valuationJoin 的 SELECT 中，只需补充筛选条件
AND db.ps_ttm >= ${minPsTtm}
AND db.ps_ttm <= ${maxPsTtm}
```

### 5.4 布林带信号

在 technicalFactorJoin 中已有 boll_upper、boll_mid、boll_lower，配合 daily close：

```sql
-- above_upper: 收盘价突破上轨
WHERE d.close > stf.boll_upper

-- below_lower: 收盘价跌破下轨
WHERE d.close < stf.boll_lower

-- squeeze: 布林带缩口 (上下轨差值 < 中轨的 5%)
WHERE (stf.boll_upper - stf.boll_lower) < stf.boll_mid * 0.05
```

### 5.5 均线趋势

利用 stock_technical_factors 已有数据，在 LATERAL JOIN 中额外取 close 近期均值：

```sql
-- bullish: 短期均线在长期上方（使用 close 做近似：当日 close > ma20 中轨）
-- 简化实现：使用 boll_mid 作为 MA20 近似值
WHERE d.close > stf.boll_mid AND stf.macd_dif > 0

-- bearish: 反之
WHERE d.close < stf.boll_mid AND stf.macd_dif < 0
```

### 5.6 北向资金筛选

```sql
-- northboundOnly = true 时
INNER JOIN LATERAL (
  SELECT vol, ratio
  FROM hk_hold_detail
  WHERE ts_code = sb.ts_code
  ORDER BY trade_date DESC LIMIT 1
) hk ON true
WHERE hk.vol > 0
```
