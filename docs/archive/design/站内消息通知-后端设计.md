# 站内消息通知系统 - 后端设计

## 概述

基于 WebSocket 的站内消息推送机制，支持通知持久化、已读状态管理和按类型偏好设置。

---

## 架构

- **NotificationModule**：`src/apps/notification/`
- **依赖**：PrismaService、EventsGateway（WebsocketModule）
- **Prisma 模型**：`Notification`（持久化通知）、`NotificationPreference`（按类型偏好开关）
- **集成点**：`PriceAlertService` 在价格预警触发时 fire-and-forget 创建通知

---

## Prisma 模型

### Notification

| 字段      | 类型              | 说明                      |
| --------- | ----------------- | ------------------------- |
| id        | Int (PK)          | 自增主键                  |
| userId    | Int               | 外键 → User               |
| type      | NotificationType  | 通知类型（枚举）          |
| title     | VarChar(128)      | 通知标题                  |
| body      | VarChar(512)      | 通知内容                  |
| data      | Json              | 附加载荷（默认 {}）       |
| isRead    | Boolean           | 是否已读（默认 false）    |
| readAt    | DateTime?         | 标记已读时间              |
| createdAt | DateTime          | 创建时间                  |

索引：`(userId, isRead, createdAt DESC)`, `(userId, createdAt DESC)`

### NotificationPreference

| 字段      | 类型              | 说明                      |
| --------- | ----------------- | ------------------------- |
| id        | Int (PK)          | 自增主键                  |
| userId    | Int               | 外键 → User               |
| type      | NotificationType  | 通知类型                  |
| enabled   | Boolean           | 是否启用该类型（默认 true）|

唯一约束：`(userId, type)`

### NotificationType 枚举

| 值                | 说明                |
| ----------------- | ------------------- |
| PRICE_ALERT       | 价格预警触发        |
| MARKET_ANOMALY    | 市场异动监控        |
| SCREENER_ALERT    | 选股订阅命中        |
| SIGNAL_TRIGGERED  | 交易信号触发        |
| SYSTEM            | 系统通知            |

## API 端点（均为 POST）

所有端点需 JWT 鉴权，操作当前登录用户的数据。

| 端点                              | 说明                      |
| --------------------------------- | ------------------------- |
| `POST /notification/list`         | 分页获取通知，支持仅未读  |
| `POST /notification/unread-count` | 获取未读数                |
| `POST /notification/mark-read`    | 标记指定通知已读          |
| `POST /notification/mark-all-read`| 标记全部已读              |
| `POST /notification/delete`       | 删除指定通知              |
| `POST /notification/preferences`  | 获取通知偏好列表          |
| `POST /notification/preferences/update` | 更新通知偏好        |

---

## WebSocket 推送

通知创建时通过 `EventsGateway.emitToUser(userId, 'notification', payload)` 推送。

前端监听 `notification` 事件，payload 结构：

```json
{
  "id": 1,
  "type": "PRICE_ALERT",
  "title": "价格预警触发：平安银行",
  "body": "000001.SZ PRICE_ABOVE 条件已触发，当前值 13.5",
  "data": { ... },
  "isRead": false,
  "createdAt": "2026-04-15T10:00:00.000Z"
}
```

---

## 业务逻辑

- 创建通知前检查 `NotificationPreference`，若用户关闭该类型则跳过（不写库不推送）
- `getPreferences()` 返回所有类型，未配置的类型默认 `enabled=true`
- `markRead()` 幂等：已读状态下不重复 update
- `deleteNotification()` 会验证通知归属（userId 必须匹配）

---

## 集成扩展点

其他 Service 通过注入 `NotificationService` 并调用 `create(input)` 触发通知，建议 fire-and-forget（`void this.notificationService.create(...)`）以避免影响主业务流。

已集成：`PriceAlertService`
可扩展：`MarketAnomalyService`、`ScreenerSubscriptionService`、`SignalService`
