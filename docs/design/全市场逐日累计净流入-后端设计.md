# 全市场逐日净流入趋势 — 后端设计

> 目标：将 `getMoneyFlowTrend` 的数据源从东财 `moneyflow_mkt_dc` 改为从个股 `moneyflow` 实时聚合，对外暴露逐日全市场净流入序列（`net_mf_amount` 全口径，非主力）。**不新建表，不新增 sync plan**，依赖现有缓存吸收聚合开销。

---

## 2. 性能评估：实时聚合 vs. 预计算表

| 维度         | 分析                                                                                                                           |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------ |
| **数据规模** | A 股约 5,200 只 × 120 日 ≈ **62 万行**                                                                                         |
| **查询**     | `WHERE trade_date IN [120 trading days] GROUP BY trade_date SUM(net_mf_amount)`                                                |
| **索引**     | `stock_capital_flows` 上有 `@@index([tradeDate])`，PostgreSQL 做 index range scan + hash aggregate                             |
| **首次耗时** | 约 **100–300 ms**                                                                                                              |
| **缓存**     | `getMoneyFlowTrend` 套了 `rememberMarketCache`，TTL = **4 小时**，实际 DB 每 4 小时才触发一次                                  |
| **结论**     | **实时聚合完全满足要求**，预计算表的维护成本（schema、migration、sync plan、aggregator service、级联修补）远超收益，不建立新表 |

## 3. 接口改造

### 3.1 改造 `getMoneyFlowTrend`

[src/apps/market/market.service.ts](src/apps/market/market.service.ts) 中的 `getMoneyFlowTrend`：

- **改前**：查 `moneyflowMktDc`，取东财大盘口径的 `netAmount`，在内存中循环累加 `cumulativeNet`。
- **改后**：查 `moneyflow`，按 `tradeDate GROUP BY` SUM `netMfAmount`，返回逐日净流入序列（万元 × 10000 转元）。去掉 `cumulativeNet`（前端如需累计曲线，对返回的序列在前端累加即可，或由 service 在内存中顺序累加后返回，与现有行为完全一致）。

```ts
// 伪代码
const rows = await this.prisma.$queryRaw<...>`
  SELECT trade_date, SUM(net_mf_amount) * 10000 AS net_amount
  FROM stock_capital_flows
  WHERE trade_date >= ${startDate}
  GROUP BY trade_date
  ORDER BY trade_date ASC
`
// 在内存中顺序累加 cumulativeNet（与现在行为相同，只是数据源变了）
```

返回结构不变（`tradeDate`、`netAmount`、`cumulativeNet`），前端零感知。

### 3.2 口径说明（需告知前端/文档）

| 字段            | 旧口径                     | 新口径                                  |
| --------------- | -------------------------- | --------------------------------------- |
| `netAmount`     | 东财大盘 `netAmount`（元） | 个股 `SUM(net_mf_amount) × 10000`（元） |
| `cumulativeNet` | 东财累计                   | 个股聚合累计                            |

两者通常相近但不完全一致（东财有自有清洗规则）。新口径与 `getMarketMoneyFlow` 单日快照完全对齐，语义更清晰（全市场所有股票净买入之和，非主力、非东财口径）。

### 3.3 不影响的接口

- `getMarketMoneyFlow` — 已经是实时 `moneyflow.aggregate`，不动。
- `getMainFlowRanking` — 主力口径，不动。
- `getSectorFlow*` / `getHsgt*` / `stock-moneyflow.service.ts` — 各自维度，不动。
