## 三、Prisma Schema 设计

新建文件 `prisma/portfolio.prisma`：

```prisma
/// 虚拟投资组合
model Portfolio {
  id          String   @id @default(cuid())
  userId      Int
  name        String   @db.VarChar(100)
  description String?  @db.VarChar(500)
  initialCash Decimal  @db.Decimal(18, 4)   /// 初始本金（元）
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  user         User                @relation(fields: [userId], references: [id], onDelete: Cascade)
  holdings     PortfolioHolding[]
  riskRules    PortfolioRiskRule[]
  violations   RiskViolationLog[]

  @@map("portfolios")
}

/// 组合持仓（每条记录代表当前持有某只股票的状态）
model PortfolioHolding {
  id          String   @id @default(cuid())
  portfolioId String
  tsCode      String   @db.VarChar(15)       /// 股票代码（如 000001.SZ）
  stockName   String   @db.VarChar(60)        /// 股票名称（冗余，避免关联查询）
  quantity    Int                              /// 持股数量（股，非手）
  avgCost     Decimal  @db.Decimal(12, 4)     /// 平均持仓成本（元/股）
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  portfolio   Portfolio @relation(fields: [portfolioId], references: [id], onDelete: Cascade)

  @@unique([portfolioId, tsCode])
  @@index([portfolioId])
  @@map("portfolio_holdings")
}

/// 风控规则类型
enum PortfolioRiskRuleType {
  MAX_SINGLE_POSITION    /// 单票仓位上限（占组合市值比例，0~1）
  MAX_INDUSTRY_WEIGHT    /// 行业集中度上限（占组合市值比例，0~1）
  MAX_DRAWDOWN_STOP      /// 最大回撤止损线（组合净值跌幅，0~1）
}

/// 组合绑定的风控规则
model PortfolioRiskRule {
  id          String                  @id @default(cuid())
  portfolioId String
  ruleType    PortfolioRiskRuleType
  threshold   Float                    /// 触发阈值，含义见 ruleType 注释
  isEnabled   Boolean                  @default(true)
  memo        String?                  @db.VarChar(200)
  createdAt   DateTime                 @default(now())
  updatedAt   DateTime                 @updatedAt

  portfolio   Portfolio    @relation(fields: [portfolioId], references: [id], onDelete: Cascade)
  violations  RiskViolationLog[]

  @@unique([portfolioId, ruleType])  /// 每组合每类型只允许一条规则
  @@map("portfolio_risk_rules")
}

/// 风控违规记录
model RiskViolationLog {
  id          String               @id @default(cuid())
  portfolioId String
  ruleId      String
  ruleType    PortfolioRiskRuleType
  actualValue Float                 /// 实测值（如实际仓位 0.35）
  threshold   Float                 /// 规则阈值（如 0.2）
  detail      String?               @db.Text   /// 详细描述（如 "000001.SZ 占比 35%"）
  checkedAt   DateTime              @default(now())

  portfolio   Portfolio        @relation(fields: [portfolioId], references: [id], onDelete: Cascade)
  rule        PortfolioRiskRule @relation(fields: [ruleId], references: [id], onDelete: Cascade)

  @@index([portfolioId, checkedAt])
  @@map("risk_violation_logs")
}
```

> **注意**：需在 `prisma/user.prisma` 中的 `User` Model 补充反向关系：
>
> ```prisma
> portfolios  Portfolio[]
> ```

### 4.2 接口规格

#### `POST /portfolio/create`

- **权限**：需登录（`@CurrentUser`）
- **输入**：`CreatePortfolioDto`
- **逻辑**：`userId` 取自 JWT，持久化至 `portfolios` 表
- **返回**：`{ id, name, initialCash, createdAt }`

#### `POST /portfolio/list`

- **输入**：（无参数，按当前用户查）
- **逻辑**：`WHERE userId = $1`，返回组合列表含每组合持仓数量统计
- **返回**：`Portfolio[]` 含 `holdingCount`

#### `POST /portfolio/detail`

- **输入**：`{ portfolioId: string }`
- **鉴权**：验证 `portfolioId.userId === currentUser.id`（下同）
- **逻辑**：
  1. 查 `portfolio_holdings`；
  2. 批量查最近交易日的 `stock_daily_valuation_metrics`（`total_mv`、`close` 用于估算当前持仓市值）；
  3. 算持仓市值、成本、浮盈、当日涨跌幅
- **返回**：`{ portfolio, holdings: HoldingDetail[], summary: PortfolioSummary }`

```typescript
type HoldingDetail = {
  tsCode: string
  stockName: string
  quantity: number
  avgCost: number
  currentPrice: number | null
  marketValue: number | null
  unrealizedPnl: number | null
  pnlPct: number | null
  weight: number | null // 占组合市值比例
  industry: string | null
}

type PortfolioSummary = {
  totalCost: number // 全部持仓成本
  totalMarketValue: number // 当前总市值
  totalUnrealizedPnl: number
  totalPnlPct: number
  cashBalance: number // initialCash - totalCost（简化：不跟踪资金流水）
}
```

#### `POST /portfolio/holding/add`

- **输入**：`AddHoldingDto`
- **逻辑**：
  - 若该股票已存在持仓：做加仓加权平均成本并更新数量
  - 若不存在：新建记录
  - 触发风控检查（Phase 3 实现后调用 `RiskCheckService.check(portfolioId)`）
- **返回**：更新后的 `PortfolioHolding`

> **加权平均成本公式**：
> `newAvgCost = (existingQty * existingAvgCost + addQty * addCost) / (existingQty + addQty)`

#### `POST /portfolio/holding/update`

- **输入**：`UpdateHoldingDto`（直接覆盖 quantity 和 avgCost，适合手动修正场景）
- **返回**：更新后的 `PortfolioHolding`

#### `POST /portfolio/holding/remove`

- **输入**：`{ holdingId: string }`
- **逻辑**：软删除（直接删除记录，无需保留历史，组合为虚拟模拟性质）

#### `POST /portfolio/pnl/today`

- **输入**：`{ portfolioId: string }`
- **逻辑**：
  1. 查全部持仓
  2. 查最近一个交易日的 `stock_daily_prices`，拿 `close` 和 `pct_chg`
  3. 算今日组合盈亏 = Σ(quantity × latestClose × pct_chg / 100)
- **返回**：`{ tradeDate, todayPnl, todayPnlPct, byHolding: TodayHoldingPnl[] }`

#### `POST /portfolio/pnl/history`

- **输入**：`PortfolioPnlHistoryDto`
- **逻辑**：Raw SQL，按交易日聚合持仓成本与当日收盘市值的关系，返回净值曲线

```sql
-- 每个交易日的组合估值（假设持仓不变，适合日线近似）
SELECT
  d.trade_date,
  SUM(h.quantity * d.close)              AS market_value,
  SUM(h.quantity * h.avg_cost)           AS cost_basis
FROM portfolio_holdings h
JOIN stock_daily_prices d
  ON d.ts_code = h.ts_code
  AND d.trade_date BETWEEN $2 AND $3  -- startDate, endDate
WHERE h.portfolio_id = $1
GROUP BY d.trade_date
ORDER BY d.trade_date;
```

返回：`{ date, marketValue, costBasis, nav: marketValue / costBasis }[]`

> **限制说明**：此简化实现假设持仓记录全程不变（取当前快照），适用于演示/研究目的。

## 五、Phase 2：组合风险分析接口

### 5.1 公共逻辑模式

所有风险分析接口共享如下前置操作：

```
1. 验证 portfolioId 归属当前用户
2. 查询 portfolio_holdings（含 tsCode、quantity、avgCost）
3. 取最近交易日日期（查 trade_calendar）
4. 批量查 stock_daily_valuation_metrics(latest_trade_date)
5. JOIN stock_basic_profiles 获取 industry / market 等分类字段
6. 计算各维度聚合结果
```

### 5.2 接口规格

#### `POST /portfolio/risk/industry`

- **功能**：行业分布（按市值）
- **输出**：

```typescript
type IndustryDistribution = {
  industry: string // 行业名（申万一级，来自 StockBasic.industry）
  stockCount: number
  totalMarketValue: number
  weight: number // 占组合总市值比例（0~1）
  items: { tsCode; name; weight }[]
}[]
```

**SQL 逻辑**（Raw SQL）：

```sql
WITH latest AS (
  SELECT DISTINCT ON (ts_code) ts_code, close, total_mv
  FROM stock_daily_valuation_metrics
  WHERE trade_date = $2    -- latest_trade_date
),
holdings AS (
  SELECT h.ts_code, h.quantity, h.avg_cost,
         b.name, b.industry,
         COALESCE(l.close * h.quantity, h.avg_cost * h.quantity) AS mkt_val
  FROM portfolio_holdings h
  JOIN stock_basic_profiles b ON b.ts_code = h.ts_code
  LEFT JOIN latest l ON l.ts_code = h.ts_code
  WHERE h.portfolio_id = $1
),
total AS (SELECT SUM(mkt_val) AS total_mv FROM holdings)
SELECT
  industry,
  COUNT(*) AS stock_count,
  SUM(mkt_val) AS total_market_value,
  SUM(mkt_val) / NULLIF((SELECT total_mv FROM total), 0) AS weight
FROM holdings
GROUP BY industry
ORDER BY weight DESC;
```

#### `POST /portfolio/risk/position`

- **功能**：个股集中度风险
- **输出**：

```typescript
type PositionConcentration = {
  tsCode: string
  name: string
  marketValue: number
  weight: number // 占组合总市值比例
  costBasis: number // 成本
  unrealizedPnl: number
  isRisky: boolean // 若有 MAX_SINGLE_POSITION 规则，weight > threshold 则 true
}[]
// + { top3Weight, top5Weight, herfindahlIndex }  // HHI 越高越集中
```

> **HHI（赫芬达尔指数）**: `HHI = Σ(weight_i²)`，越接近 1 越集中

#### `POST /portfolio/risk/marketcap`

- **功能**：市值分布（按规模段划分）
- **规模段划分**：小盘 < 50 亿 ≤ 中盘 < 200 亿 ≤ 大盘
- **数据来源**：`stock_daily_valuation_metrics.total_mv`（亿元）
- **输出**：

```typescript
type MarketCapDistribution = {
  tier: 'small' | 'mid' | 'large' // < 50亿 / 50~200亿 / > 200亿
  label: string // "小盘" / "中盘" / "大盘"
  stockCount: number
  totalMarketValue: number
  weight: number
}[]
```

#### `POST /portfolio/risk/beta`

- **功能**：组合加权 Beta 暴露（相对沪深300）
- **基准指数代码**：`399300.SZ`（沪深300，来自 `IndexDaily`）
- **计算窗口**：近 250 个交易日收益率序列
- **Beta 计算（TypeScript 服务层）**：

```typescript
// 单票 Beta = Cov(r_stock, r_index) / Var(r_index)
// 组合 Beta = Σ(weight_i × beta_i)

function calcBeta(stockReturns: number[], indexReturns: number[]): number {
  const n = Math.min(stockReturns.length, indexReturns.length)
  const meanS = mean(stockReturns.slice(-n))
  const meanI = mean(indexReturns.slice(-n))
  const cov = stockReturns.slice(-n).reduce((sum, r, i) => sum + (r - meanS) * (indexReturns[i] - meanI), 0) / n
  const varI = indexReturns.slice(-n).reduce((sum, r) => sum + (r - meanI) ** 2, 0) / n
  return varI === 0 ? 1 : cov / varI
}
```

- **输出**：

```typescript
type BetaAnalysis = {
  portfolioBeta: number // 加权 Beta
  benchmarkCode: string // 399300.SZ
  window: number // 实际使用交易日数
  byHolding: {
    tsCode: string
    name: string
    beta: number
    weight: number
    contribution: number // = weight × beta
  }[]
}
```

> **注意**：新上市股票历史数据不足 250 日时，使用实际可用天数（至少 60 日才参与计算，否则跳过并返回 null beta）。

## 六、Phase 3：风控规则引擎

### 6.1 规则类型说明

| ruleType              | threshold 含义                  | 检查时机          | 示例          |
| --------------------- | ------------------------------- | ----------------- | ------------- |
| `MAX_SINGLE_POSITION` | 单票占组合总市值最大比例（0~1） | 调仓后触发        | `0.20`（20%） |
| `MAX_INDUSTRY_WEIGHT` | 行业占组合总市值最大比例（0~1） | 调仓后触发        | `0.40`（40%） |
| `MAX_DRAWDOWN_STOP`   | 组合净值最大回撤止损线（0~1）   | 调仓后 + 手动触发 | `0.15`（15%） |

### 6.2 接口规格

#### `POST /portfolio/risk/rules/delete`

- **输入**：`{ ruleId: string }`

#### `POST /portfolio/risk/check`

- **输入**：`{ portfolioId: string }`
- **功能**：手动触发风控检查，返回所有规则的当前实测值与是否违规
- **逻辑**：

```
1. 查组合全部 isEnabled=true 的规则
2. 取最新持仓市值（同 Phase 2 公共逻辑）
3. 对每条规则逐一检测：
   - MAX_SINGLE_POSITION：找最大单票 weight
   - MAX_INDUSTRY_WEIGHT：找最大行业 weight
   - MAX_DRAWDOWN_STOP：查近期 pnl/history，算最大回撤
4. 将所有违规记录写入 risk_violation_logs（即使本次已存在近期记录也要插入新条目）
5. 返回全部规则检查结果
```

- **返回**：

```typescript
type RiskCheckResult = {
  portfolioId: string
  checkedAt: string
  rules: {
    ruleId: string
    ruleType: PortfolioRiskRuleType
    threshold: number
    actualValue: number
    isViolated: boolean
    detail: string
  }[]
  violationCount: number
}
```

**钩子集成**：在 `PortfolioHoldingService.addHolding` 和 `updateHolding` 末尾，注入 `RiskCheckService` 并调用检查（仅检查 `MAX_SINGLE_POSITION` 和 `MAX_INDUSTRY_WEIGHT` 两类，`MAX_DRAWDOWN_STOP` 需历史数据较重，调仓时不自动触发）。

#### `POST /portfolio/risk/violations`

- **输入**：`{ portfolioId: string; limit?: number }`（默认 `limit=50`）
- **返回**：`RiskViolationLog[]`，按 `checkedAt DESC` 排序
