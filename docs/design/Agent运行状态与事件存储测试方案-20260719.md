## 3. 业务理解

### 3.1 业务场景

- 用户发送研究请求后，数据库保存可恢复 Run；Worker 领取 lease 并推进状态。
- 页面刷新或 SSE 断线后，服务按用户范围和 sequence 从 PostgreSQL 无重无漏重放。
- 用户取消与 Worker 完成并发时，只允许一个 CAS 胜出；取消生效后迟到完成不能覆盖终态。
- Worker 崩溃后，新 Worker 只能接管已过期 lease，并从结构化 checkpoint 恢复。

### 3.2 核心业务规则

| 规则编号 | 规则 | 来源 | 测试影响 |
| --- | --- | --- | --- |
| R-001 | Run 只允许 `QUEUED→RUNNING/CANCELLED`、`RUNNING→CANCEL_REQUESTED/COMPLETED/FAILED`、`CANCEL_REQUESTED→CANCELLED` | 公共状态模型 | 表驱动覆盖全部合法/非法转换 |
| R-002 | `COMPLETED/FAILED/CANCELLED` 为不可离开终态；终态事件是最后一条业务事件 | 状态模型、SSE 协议 | 终态后 append/late completion 必须拒绝 |
| R-003 | 每次状态转换用 expected `statusVersion` CAS；成功后版本加一并同事务追加事件 | Batch 005 | 冲突不产生半状态或孤立事件 |
| R-004 | 同一 Run 只有一个有效 lease；过期后可接管，旧 owner 不能 heartbeat 或提交状态 | Batch 005 | 双 Worker 与 takeover 并发测试 |
| R-005 | sequence 从 1 严格递增且唯一；并发 append 不允许重复或缺口 | SSE 协议、数据库设计 | 20 路并发后应为连续集合 |
| R-006 | replay 必须先校验 `userId + runId` owner，并只返回 `sequence > afterSequence` | Batch 005 | 越权返回不可枚举结果；分页有序 |
| R-007 | checkpoint/event payload 只存结构化公开状态或引用 ID，经过脱敏与大小限制 | 安全要求 | secret/hidden reasoning 不得持久化 |
| R-008 | Step attempt 唯一 `(runId,stepKey,attempt)`，重复恢复不得重复完成副作用 | Batch 005 | 重复插入受唯一约束保护 |
| R-009 | `AiToolCall/AiModelCall` 关联有效 Run/Step；删除 owner 不产生 orphan audit | Batch 005 | FK 与删除策略测试 |
| R-010 | PostgreSQL 是唯一权威源；Redis 不可用不影响状态和 replay | Batch 005 | 集成测试不启动 Redis |

### 3.3 状态变化与不变量

| 类型 | 说明 |
| --- | --- |
| 数据写入 | 状态、版本、lease/checkpoint 和事件按业务事务原子写入 |
| 缓存/队列/通知 | 本批不写 Redis/队列；publish 失败不得回滚已提交权威事件 |
| 权限边界 | 用户方法强制 owner scope；Worker 方法要求 lease owner |
| 时间规则 | lease 使用 PostgreSQL `clock_timestamp()`；应用机器时钟不决定接管资格 |
| 数据不变量 | 非负 version/attempt/sequence/budget；终态时间存在；事件 payload 带 `schemaVersion` |

## 4. 内部接口契约

| 接口 | 输入 | 输出 | 权限 | 备注 |
| --- | --- | --- | --- | --- |
| `createRun` | owner、conversation/message、版本、预算、幂等键 | `QUEUED` Run | 用户范围 | 同 user/clientRequestId 同 hash 幂等 |
| `claimRun` | runId、workerId、leaseMs | lease + Run | Worker | 仅 QUEUED 或已过期非终态 lease |
| `heartbeat` | runId、workerId、leaseMs | 新 expiry | lease owner | 失租立即 conflict |

| `requestCancel` | userId、runId、expectedVersion | 新/既有终态 Run | owner | 终态与重复取消幂等 |
| `replay` | userId、runId、afterSequence、limit | 有序 Event[] | owner | `sequence > cursor` |
