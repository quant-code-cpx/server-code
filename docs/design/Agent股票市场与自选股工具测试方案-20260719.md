## 3. 业务理解

### 3.1 业务场景

- 用户用名称、代码或简称解析证券；歧义时收到候选并澄清，不由模型猜测。
- 用户查询有明确周期、复权、日期、字段和单位的价格序列，用于研究和后续确定性计算。
- 用户批量读取股票概览、市场截面和板块归属；每个数据源显示自己的真实截止日。
- 用户读取自己的自选组；身份只来自服务端 context，不能读取他人组或让缓存串租户。

### 3.2 核心业务规则

| 规则编号 | 规则                                                                                         | 来源                       | 测试影响                                         |
| -------- | -------------------------------------------------------------------------------------------- | -------------------------- | ------------------------------------------------ |
| R-001    | 六个 Tool 必须是唯一注册、READ、idempotent、strict schema                                    | Batch 007/006              | 非法字段和未授权调用在 Facade 前拒绝             |
| R-002    | 证券解析最多 20 候选，按确定性得分与代码排序；近分候选不自动选择                             | internal-data schema       | exact/partial/歧义/退市分别验证                  |
| R-003    | Tool 日期使用 `YYYY-MM-DD`；领域边界才转 `YYYYMMDD`；`startDate <= endDate`                  | common schema/金融数据边界 | 非法日期、反向范围拒绝                           |
| R-004    | 行情按真实交易日升序，最多 5,000 bars；超限返回最近 N 条并标 `truncated`                     | Batch 007                  | 分页边界和稳定排序                               |
| R-005    | QFQ 价格乘 `factor/latestFactor`；HFQ 使用经验证后复权口径；日周月 `pctChange` 统一百分数    | Batch 000/金融数据边界     | 可手算 golden case，禁止 100 倍单位错误          |
| R-006    | `vol` 单位为手、`amount` 为千元、比例为百分数；数据库 `null` 保持 `null`                     | 数据能力盘点               | 不把缺失值补 0                                   |
| R-007    | 历史 overview 不得混入 requested as-of 之后的行情/估值/快报                                  | 金融数据边界               | historical fixture 只返回截止日前数据            |
| R-008    | Market 每个 section 独立 `asOf/status`；缺数据不能借其他 section 日期伪装已更新              | internal-data schema       | 混合水位与空 section                             |
| R-009    | 申万/指数按不晚于 effectiveDate 的有效期或快照过滤；THS 概念缺历史有效期                     | 数据能力盘点               | 显式历史概念查询质量失败，当前查询带限制 warning |
| R-010    | Watchlist userId 只从 ToolAccessContext 注入；不存在与非本人同一外部语义                     | 安全设计                   | A/B 用户、伪造 userId、管理员角色全部验证        |
| R-011    | Watchlist 无 ID 时按组/成员 sortOrder 稳定返回；`limit` 是跨组成员总上限                     | 本批默认决策               | 空组、跨组截断和 quote 开关                      |
| R-012    | 公共市场缓存 key 覆盖日期/sections/sectorType/topN/schema version；私有 Watchlist 本批不缓存 | Batch 007/安全设计         | 不同请求不串数据，私有调用始终 owner query       |

### 3.3 状态变化与不变量

| 类型            | 说明                                                                     |
| --------------- | ------------------------------------------------------------------------ |
| 数据写入        | 六个 Tool 只读；除公共缓存外不写业务表，不触发补数或同步                 |
| 缓存/队列/通知  | 公共 market snapshot 可短缓存；Watchlist 不缓存；无队列/通知             |
| 权限边界        | 公共金融数据仍需认证；私有数据查询始终包含 `userId` 和可选 `watchlistId` |
| 时间/交易日规则 | requested date 解析为不晚于该日的真实交易日；section 可有不同水位        |
| 数据不变量      | 稳定排序、固定 select、数值/null 不变形、来源 Model 真实、结果有界       |

## 6. 风险分析

| 风险                       | 业务影响                     | 测试策略                             | 优先级 |
| -------------------------- | ---------------------------- | ------------------------------------ | ------ |
| QFQ 方向或周月收益单位错误 | 图表、收益和后续研究数字错误 | 独立手算 factor/OHLC golden case     | P0     |

| 大范围行情/成分查询        | DB 压力、模型上下文膨胀      | limit、truncation、EXPLAIN、负载测试 | P1     |
