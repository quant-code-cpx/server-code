## 2. 核心业务规则

| 规则编号 | 规则 | 来源 | 测试影响 |
| -------- | ---- | ---- | -------- |
| R-TA01 | 全部端点需 SUPER_ADMIN 角色 | @UseGuards(RolesGuard) @Roles(UserRole.SUPER_ADMIN) | 非 SUPER_ADMIN 应 403 |
| R-TA02 | 全部端点需 JWT 认证 | JwtAuthGuard (全局) | 无 Token 应 401 |
| R-TA03 | sync 端点 mode 必须是 incremental/full | ManualSyncDto @IsIn(TUSHARE_SYNC_MODES) | 非法值应 400 |
| R-TA04 | sync 端点 tasks 可选，但每个元素必须是 TushareSyncTaskName 枚举 | ManualSyncDto @IsEnum | 非法任务名应 400 |
| R-TA05 | sync-logs 支持按 task/status/startDate/endDate 过滤 | SyncLogQueryDto | 各字段可选 |
| R-TA06 | sync-logs page >= 1, pageSize 1~100 | SyncLogQueryDto @Min(1) @Max(100) | 超范围应 400 |
| R-TA07 | sync-logs startDate/endDate 必须是 ISO 8601 | SyncLogQueryDto @IsISO8601 | 格式错误应 400 |
| R-TA08 | retry-queue pageSize 上限 100，代码内 Math.min | Controller 逻辑 | > 100 自动截断为 100 |
| R-TA09 | sync 端点返回 HTTP 202 (ACCEPTED) | @HttpCode(HttpStatus.ACCEPTED) | 状态码 202 |
| R-TA10 | quality/check 端点返回 HTTP 202 | @HttpCode(HttpStatus.ACCEPTED) | 状态码 202 |
| R-TA11 | quality/gaps dataSet 必填 | Controller @Body() dto: { dataSet: string } | 缺失时服务层应处理 |
| R-TA12 | sync-status-overview refresh 可选，默认使用缓存 | Controller 逻辑 | refresh=true 强制刷新 |
