# 智能体事件流运行手册

## 1. 接口

`POST /api/agent/runs/events` 使用 Bearer JWT 和 JSON Body：

```json
{ "runId": "run_...", "afterSequence": 0 }
```

可选 Header `Last-Event-ID` 填持久 `eventId`。Header 存在时优先于 `afterSequence`，且只能解析当前 Body `runId` 的 USER 事件。跨租户 Run 返回 404/code 6002；非法或跨 Run eventId 返回 400/code 9001。

成功响应为 raw `text/event-stream`，不经过统一 JSON wrapper：

```text
id: evt_...
event: agent.progress
retry: 3000
data: {"schemaVersion":"1.0","eventId":"evt_...","sequence":7,...}

```

## 2. 正确性边界

- PostgreSQL `AiRunEvent` 是唯一权威源；Redis 不保存 replay 历史。
- owner 校验和 Last-Event-ID 解析完成后才写 SSE headers。
- 连接先固定 high-water mark、分页重放到该点，再 tail 新事件；数据库提交前不会发送。
- 多实例通过 PostgreSQL polling 看到相同持久事件；通知丢失或 Redis 故障不丢事件。
- 客户端断开只终止订阅，不取消 Run。收到 `agent.completed/failed/cancelled` 后服务端结束流。
- heartbeat 为 `: heartbeat` comment，不占 sequence、不落库。

## 3. 配置

```dotenv
AGENT_SSE_HEARTBEAT_MS=15000
AGENT_SSE_IDLE_TIMEOUT_MS=300000
AGENT_SSE_MAX_CONNECTIONS_PER_USER=3
AGENT_SSE_MAX_BUFFER_BYTES=1048576
```

- heartbeat 范围 1–60 秒。
- idle timeout 范围 10 秒–1 小时，且必须大于 heartbeat。
- 每用户连接上限范围 1–20，当前按 API 进程计数；全局租户级限额留给生产网关/Batch 026。
- buffer 上限范围 16 KiB–16 MiB。单帧超限、`writableLength` 超限或一个 heartbeat 周期内不 drain，均按慢消费者断开。

响应固定包含 `Cache-Control: no-cache, no-transform` 和 `X-Accel-Buffering: no`。开发 Compose 直接暴露 Nest 端口，无额外代理缓冲；生产代理仍需在 Batch 026 显式关闭 buffering。

## 4. 数据迁移

Batch 014 不新增表，但包含三条一次性数据修复：

- `20260720050000_compact_agent_user_event_sequences`：删除旧 `run.cancel_requested` INTERNAL 事件并压实 sequence。
- `20260720051000_remove_agent_event_payload_schema_version`：移除错误写入 payload 的顶层 `schemaVersion`。
- `20260720052000_align_agent_stream_event_payloads`：回填 `agent.started` 的 workflow 字段，并从 `AiModelCall` 恢复 completed token usage。

迁移通过 `prisma migrate deploy` 自动执行；不要用 `db push` 替代。

## 5. 指标

- `agent_sse_active_connections`
- `agent_sse_connections_total{result}`
- `agent_sse_events_total{phase="replay|live"}`
- `agent_sse_bytes_total`
- `agent_sse_duration_seconds{reason}`
- `agent_sse_replay_lag_events`
- `agent_sse_disconnects_total{reason}`

HTTP latency 在 response finish/close 时结算，不在 Controller 返回时提前结算。日志只记录 runId、reason、耗时和安全错误，不记录 model delta 或 Tool 正文。

## 7. 故障判断

- 200 但无 frame：查 `AgentStreamController` 的安全错误日志，重点检查旧事件 payload 是否已完成三条 migration。
- sequence gap：停止发布前端，检查是否仍存在 INTERNAL 事件占用公共 sequence，禁止客户端静默跳过。
- 频繁 `slow_consumer`：检查反向代理 buffering、客户端 reader 是否及时 drain、单帧大小和 buffer 配置。
- replay 正常但 live 延迟：查 PostgreSQL 查询耗时和连接数；正确性不依赖 Redis。
- 429：同一用户在当前 API 实例达到连接上限；关闭旧页面/旧 AbortController 后重连。
- Token 到期：服务端关闭流；前端刷新 Token 后用最后确认 eventId 新建 POST 请求。
