## 2. 模块循环策略

### 2.1 循环顺序

1. **Auth（认证）**：登录安全边界与会话正确性（高风险）

## 3. 模块一：Auth（认证）

### 3.1 核心规则

- `POST /auth/captcha`：60s 一次性验证码；限流 10/min
- `POST /auth/login`：限流 5/min，成功返回 `accessToken`
- `POST /auth/refresh`：支持 body/cookie 获取 refreshToken；旧 RT 有 10s 宽限
- `POST /auth/logout`：黑名单 AT + 撤销 RT + 清 cookie
- 常量：`LOGIN_MAX_FAIL=5`、`LOGIN_LOCK_DURATION=600`、`CAPTCHA_TTL=60`

### 3.2 风险与不变量

- P0：`LoginDto` 使用 `@Allow()`，空 body 可能穿透验证层
- P0：刷新/登出令牌语义（缺 token 时错误码）
- P1：验证码大小写、过期、重放边界

### 3.3 用例矩阵（V2）

| 用例ID | 类型 | 优先级 | 场景 | 期望 |
| --- | --- | --- | --- | --- |
| AUTH-V2-001 | BIZ | P0 | `/auth/captcha` 正常 | 201 + `captchaId/svgImage` |
| AUTH-V2-002 | BIZ | P0 | `/auth/login` 正常体 | 201 + `accessToken` |
| AUTH-V2-003 | ERR | P0 | `/auth/login` 空 body | **应为 400**（若非 400 记缺陷） |
| AUTH-V2-004 | ERR | P0 | `/auth/refresh` 无 token | 401 `INVALID_REFRESH_TOKEN` |
| AUTH-V2-005 | SEC | P1 | `/auth/logout` 仅 header/cookie 缺失 | 不应 500，接口可幂等返回 |

## 4. 模块二：Event-study（事件驱动研究）

### 4.1 核心规则

- `POST /event-study/analyze` 默认窗口：`preDays=5, postDays=20`
- `POST /event-study/signal-rules/scan`：仅 `SUPER_ADMIN`
- `POST /event-study/events`：分页与事件类型枚举严格校验

### 4.2 风险与不变量

- P0：管理员接口越权
- P0：分析参数边界（窗口最小/最大）
- P0：BigInt 序列化风险（历史问题点）

### 4.3 用例矩阵（V2）

| 用例ID | 类型 | 优先级 | 场景 | 期望 |
| --- | --- | --- | --- | --- |
| EST-V2-001 | BIZ | P0 | `/event-study/event-types/list` | 201 + 非空数组 |
| EST-V2-002 | ERR | P0 | `/event-study/events` 无效 `eventType` | 400 |
| EST-V2-003 | SEC | P0 | 普通用户调用 `/signal-rules/scan` | 403 |
| EST-V2-004 | DATA | P0 | `/events` 返回含 BigInt 字段 | 不应触发 500 序列化异常 |
| EST-V2-005 | EDGE | P1 | `analyze` 最小窗口 | 201 且结构完整 |

## 5. 模块三：Alert（预警与监控）

### 5.1 核心规则

- `calendar/list` 日期格式 `YYYYMMDD`
- `price-rules/scan` 与 `anomalies/scan`：仅 `ADMIN`
- `anomalies/detail` body 使用 `anomalyId`
- `limit-list` `pageSize<=200`，超限应校验/截断

### 5.2 风险与不变量

- P0：管理员端点越权
- P0：`anomalies/detail` 字段名不匹配导致 `undefined` 风险
- P1：日期/分页参数校验边界

### 5.3 用例矩阵（V2）

| 用例ID | 类型 | 优先级 | 场景 | 期望 |
| --- | --- | --- | --- | --- |
| ALT-V2-001 | BIZ | P0 | `/alert/calendar/list` 基础查询 | 201 + 列表结构 |
| ALT-V2-002 | ERR | P0 | `/alert/calendar/list` 非法日期 | 400 |
| ALT-V2-003 | SEC | P0 | 非 ADMIN 调 `/alert/anomalies/scan` | 403 |

## 6. 模块四：Calendar（独立模块）

### 6.1 核心规则

- `POST /calendar/range`：`startDate/endDate` 必填且 `YYYYMMDD`
- `POST /calendar/upcoming`：`days` 默认 30，范围 1~365
- 控制器类级 `JwtAuthGuard`，无 token 应 401

### 6.2 风险与不变量

- P0：JWT 保护是否失效
- P1：日期格式和 `days` 边界

### 6.3 用例矩阵（V2）

| 用例ID | 类型 | 优先级 | 场景 | 期望 |
| --- | --- | --- | --- | --- |
| CAL-V2-001 | SEC | P0 | 无 token 调 `/calendar/range` | 401 |
| CAL-V2-002 | BIZ | P0 | 合法 token + range 正常 | 201 |
| CAL-V2-003 | ERR | P1 | `/calendar/range` 日期格式错误 | 400 |
| CAL-V2-004 | BIZ | P1 | `/calendar/upcoming` 默认参数 | 201 |
| CAL-V2-005 | EDGE | P1 | `/calendar/upcoming` `days=366` | 400 |
