# 概念板块映射 — 后端设计

> **范围**：同花顺概念板块目录（`ths_index`）+ 概念成分映射（`ths_member`）
>
> **积分要求**：`ths_index` 和 `ths_member` 均需 **6000 积分**
>
> **目标**：新增 2 张 Prisma 表、2 个 mapper、2 个同步计划项，新增 3 个查询接口

---

## 一、接口字段确认（来源：Tushare 官方文档）

### 1.1 ths_index — 同花顺板块目录（doc_id=259）

**说明**：获取同花顺板块指数，包括概念（N）、行业（I）、地域（R）、特色（S/ST/TH/BB）指数。单次最大 5000 行，一次可提取全量，勿循环。概念板块取 `type='N'`，当前 A 股概念板块 270 条左右。

**输入参数**：

| 参数     | 类型 | 必填 | 说明                                 |
| -------- | ---- | ---- | ------------------------------------ |
| ts_code  | str  | N    | 指数代码                             |
| exchange | str  | N    | 市场类型 A-A股 / HK-港股 / US-美股   |
| type     | str  | N    | N-概念 I-行业 R-地域 S/ST/TH/BB 特色 |

**输出参数**：

| Tushare 字段 | 类型 | 必有 | 说明                          |
| ------------ | ---- | ---- | ----------------------------- |
| ts_code      | str  | Y    | 板块代码（如 885835.TI）      |
| name         | str  | Y    | 板块名称（如 参股银行）       |
| count        | int  | Y    | 成分股个数                    |
| exchange     | str  | Y    | 交易所（A / HK / US）         |
| list_date    | str  | Y    | 上市日期（YYYYMMDD）          |
| type         | str  | Y    | N-概念 I-行业 R-地域 S-特色等 |

### 1.2 ths_member — 同花顺概念成分（doc_id=261）

**说明**：获取同花顺概念板块成分，按 `ts_code`（板块代码）循环查询。单次无明确条数限制（实测单个概念通常 < 200 行）。

**输入参数**：

| 参数     | 类型 | 必填 | 说明         |
| -------- | ---- | ---- | ------------ |
| ts_code  | str  | N    | 板块指数代码 |
| con_code | str  | N    | 成分股票代码 |

**输出参数**：

| Tushare 字段 | 类型  | 必有 | 说明                   |
| ------------ | ----- | ---- | ---------------------- |
| ts_code      | str   | Y    | 板块代码               |
| con_code     | str   | Y    | 成分股票代码           |
| con_name     | str   | Y    | 成分股票名称           |
| weight       | float | N    | 权重（暂无，预留）     |
| in_date      | str   | N    | 纳入日期（暂无，预留） |
| out_date     | str   | N    | 剔除日期（暂无，预留） |
| is_new       | str   | N    | 是否最新（Y/N）        |

## 二、Prisma Schema 设计

### 2.1 ThsIndex（同花顺板块目录）

文件：`prisma/tushare_ths_index.prisma`

```prisma
// 同花顺板块目录（来自 ths_index 接口）
// 包含概念（N）、行业（I）、地域（R）等板块类型
model ThsIndex {
  tsCode    String   @id @map("ts_code") @db.VarChar(24)
  name      String   @db.VarChar(64)
  count     Int?
  exchange  String?  @db.VarChar(8)
  listDate  DateTime? @map("list_date") @db.Date
  type      String   @db.VarChar(8)           // N-概念 I-行业 R-地域 S-特色
  syncedAt  DateTime @default(now()) @map("synced_at")

  members   ThsMember[]

  @@index([type])
  @@index([exchange])
  @@index([name])
  @@map("ths_index_boards")
}
```

### 2.2 ThsMember（同花顺概念成分）

文件：`prisma/tushare_ths_member.prisma`

```prisma
// 同花顺板块成分映射（来自 ths_member 接口）
model ThsMember {
  id      Int    @id @default(autoincrement())
  tsCode  String @map("ts_code") @db.VarChar(24)   // 板块代码
  conCode String @map("con_code") @db.VarChar(16)  // 成分股票代码
  conName String? @map("con_name") @db.VarChar(64)
  isNew   String? @map("is_new") @db.VarChar(4)    // Y-最新成分 N-历史成分
  syncedAt DateTime @default(now()) @map("synced_at")

  board   ThsIndex @relation(fields: [tsCode], references: [tsCode])

  @@unique([tsCode, conCode])
  @@index([tsCode])
  @@index([conCode])
  @@index([conCode, isNew])
  @@map("ths_index_members")
}
```

### 3.6 BasicSyncService 同步计划

**策略**：

- `THS_INDEX`：一次全量 API 调用获取所有板块，`bootstrapEnabled: true`，每周日早晨刷新（概念更新频率低）
- `THS_MEMBER`：先查 DB 获取概念板块代码列表，再逐个调用 `ths_member`，全量 upsert，`bootstrapEnabled: true`，同样每周一次

**频控说明**：`ths_member` 需按概念代码循环调用（约 270 次），请求间需加 300ms 节流（与 `index_member_all` 循环策略一致）。

```typescript
{
  task: TushareSyncTaskName.THS_INDEX,
  label: '同花顺板块目录',
  category: 'basic',
  order: 70,
  bootstrapEnabled: true,
  supportsManual: true,
  supportsFullSync: true,
  requiresTradeDate: false,
  schedule: {
    cron: '0 30 9 * * 0',          // 每周日 09:30
    timeZone: this.helper.syncTimeZone,
    description: '每周日刷新同花顺板块目录',
  },
  execute: () => this.syncThsIndex(),
},
{
  task: TushareSyncTaskName.THS_MEMBER,
  label: '同花顺概念成分',
  category: 'basic',
  order: 71,
  bootstrapEnabled: true,
  supportsManual: true,
  supportsFullSync: true,
  requiresTradeDate: false,
  schedule: {
    cron: '0 0 10 * * 0',          // 每周日 10:00（THS_INDEX 之后）
    timeZone: this.helper.syncTimeZone,
    description: '每周日刷新同花顺概念成分',
  },
  execute: () => this.syncThsMember(),
},
```

**syncThsIndex 实现摘要**：

```typescript
async syncThsIndex() {
  const raw = await this.api.getThsIndex()
  // 全量 deleteMany + createMany 刷新
  const records = raw.items.map((row) => mapThsIndexRecord(row))
  await this.helper.prisma.$transaction([
    this.helper.prisma.thsIndex.deleteMany(),
    this.helper.prisma.thsIndex.createMany({ data: records }),
  ])
  this.logger.log(`syncThsIndex: upserted ${records.length} boards`)
}
```

**syncThsMember 实现摘要**：

```typescript
async syncThsMember() {
  // 1. 从 DB 读取所有概念板块代码
  const boards = await this.helper.prisma.thsIndex.findMany({
    where: { type: 'N' },
    select: { tsCode: true },
  })

  // 2. 全量清空后重建（概念成分变动时整体替换）
  await this.helper.prisma.thsMember.deleteMany()

  // 3. 逐板块抓取并写入
  for (const board of boards) {
    const raw = await this.api.getThsMemberByCode(board.tsCode)
    const records = raw.items.map((row) => mapThsMemberRecord(row))
    if (records.length > 0) {
      await this.helper.prisma.thsMember.createMany({ data: records, skipDuplicates: true })
    }
    await this.helper.sleep(300)   // 频控节流
  }
  this.logger.log(`syncThsMember: completed for ${boards.length} concepts`)
}
```

## 五、Service 实现逻辑

### 5.1 MarketService 新增方法

```typescript
async getConceptList(dto: ConceptListDto) {
  const { keyword, page = 1, pageSize = 50 } = dto
  const where = {
    type: 'N',
    ...(keyword ? { name: { contains: keyword } } : {}),
  }
  const [total, items] = await Promise.all([
    this.prisma.thsIndex.count({ where }),
    this.prisma.thsIndex.findMany({
      where,
      select: { tsCode: true, name: true, count: true, listDate: true },
      orderBy: { count: 'desc' },
      skip: (page - 1) * pageSize,
      take: pageSize,
    }),
  ])
  return { total, page, pageSize, items }
}

async getConceptMembers(dto: ConceptMembersDto) {
  const { tsCode, page = 1, pageSize = 100 } = dto
  const board = await this.prisma.thsIndex.findUnique({
    where: { tsCode },
    select: { name: true },
  })
  const [total, items] = await Promise.all([
    this.prisma.thsMember.count({ where: { tsCode } }),
    this.prisma.thsMember.findMany({
      where: { tsCode },
      select: { conCode: true, conName: true },
      skip: (page - 1) * pageSize,
      take: pageSize,
    }),
  ])
  return { tsCode, name: board?.name, total, items }
}
```

### 5.2 StockService 新增方法

```typescript
async getStockConcepts(dto: StockConceptsDto) {
  const { tsCode } = dto
  const memberships = await this.prisma.thsMember.findMany({
    where: { conCode: tsCode },
    include: { board: { select: { tsCode: true, name: true } } },
  })
  const stock = await this.prisma.stockBasic.findUnique({
    where: { tsCode },
    select: { name: true },
  })
  return {
    tsCode,
    name: stock?.name,
    concepts: memberships.map((m) => ({
      tsCode: m.board.tsCode,
      name: m.board.name,
    })),
  }
}
```

## 七、注意事项

1. **`ths_index` 一次全量**：无需分页，单次 call 即可拿到所有约 400+ 条（包含行业/概念/地域）。建议在同步时不过滤 `type`，全部保存入库，查询层再按 `type='N'` 筛选概念。

2. **`ths_member` 循环节流**：每个概念板块一次请求，频率不超过 200次/分钟（即 ≥ 300ms 间隔）。约 270 个 A 股概念，全量同步约 1.5 分钟内完成。

3. **全量替换策略**：`ThsMember` 按全量 `deleteMany + createMany` 刷新，不做增量，因为成分变化规律无明确时间戳可追踪。

4. **与 `moneyflow_ind_dc` 的区别**：已有的资金流向概念数据（`moneyflow_ind_dc`）是按交易日切片的资金流量。本设计是静态成分映射（哪些股票属于哪个概念），两者互补，不冲突。

5. **商业用途注意**：Tushare 文档注明同花顺数据版权归属同花顺，商业用途需联系同花顺授权。本服务仅作个人量化研究使用。
