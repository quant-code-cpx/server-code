## 一、功能总览

| 功能 | 模块路径 | 触发方式 | 依赖数据 | 需新表 |
|---|---|---|---|---|
| **事件日历** | `GET /alert/calendar` | 按需查询 | disclosure_date / share_float / dividend / forecast | 否 |
| **价格预警规则管理** | `/alert/price-rules` CRUD | 盘后定时扫描（每日 19:00） | stock_daily_prices | 是（1 张） |

**全部功能均为盘后日频触发（T+0 收盘数据），不依赖实时行情。**

## 三、Feature A：事件日历接口

### 3.1 接口设计

```
GET /alert/calendar
```

**Query 参数：**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `startDate` | `string (YYYYMMDD)` | 是 | 查询起始日期 |
| `endDate` | `string (YYYYMMDD)` | 是 | 查询截止日期（最多跨 90 天） |
| `tsCode` | `string` | 否 | 不传则返回全市场 |
| `types` | `string[]` | 否 | 筛选事件类型，默认全部。可选：`DISCLOSURE` / `FLOAT` / `DIVIDEND` / `FORECAST` |

**响应：**

```typescript
interface CalendarEventDto {
  date: string          // YYYYMMDD，事件日期
  tsCode: string
  stockName: string | null
  type: CalendarEventType
  title: string         // 简短标题，例如 "除权除息日 0.5元/股"
  detail: Record<string, unknown>  // 原始关键字段，供前端展开
}

interface CalendarResultDto {
  startDate: string
  endDate: string
  totalCount: number
  events: CalendarEventDto[]  // 按 date ASC 排序
}

enum CalendarEventType {
  DISCLOSURE = 'DISCLOSURE',  // 财报披露计划（pre_date / actual_date）
  FLOAT      = 'FLOAT',       // 限售股解禁（float_date）
  DIVIDEND   = 'DIVIDEND',    // 除权除息日（ex_date）
  FORECAST   = 'FORECAST',    // 业绩预告（ann_date）
}
```

### 3.2 数据聚合逻辑

每类事件的「代表日期」（按该日期定位到日历格子）：

| 类型 | 代表日期字段 | 筛选条件 |
|---|---|---|
| `DISCLOSURE` | `actual_date`（若为 null 则用 `pre_date`） | actual_date 或 pre_date 在区间内 |
| `FLOAT` | `float_date` | float_date 在区间内 |
| `DIVIDEND` | `ex_date` | ex_date 在区间内，`div_proc = '实施'` |
| `FORECAST` | `ann_date` | ann_date 在区间内 |

每类分别查询，然后合并排序。不使用 UNION SQL，而是 4 路并行查询后在 JS 合并，保持代码清晰可维护。

### 3.3 title 生成规则

```typescript
function buildTitle(type: CalendarEventType, detail: Record<string, unknown>): string {
  switch (type) {
    case CalendarEventType.DISCLOSURE:
      return `财报披露${detail.actualDate ? '（实际）' : '（预计）'}`
    case CalendarEventType.FLOAT:
      return `限售解禁 ${detail.floatRatio != null ? (Number(detail.floatRatio) * 100).toFixed(2) + '%' : ''}`
    case CalendarEventType.DIVIDEND:
      return `除权除息${detail.cashDiv ? ` 现金 ${detail.cashDiv}元` : ''}${detail.stkDiv ? ` 送股 ${detail.stkDiv}股` : ''}`
    case CalendarEventType.FORECAST:
      return `业绩预告（${detail.type ?? ''}）${detail.pChangeMin != null ? ` 净利润变动 ${detail.pChangeMin}%~${detail.pChangeMax}%` : ''}`
  }
}
```

### 3.4 DTO

```typescript
// alert-calendar.dto.ts
export class CalendarQueryDto {
  @Matches(/^\d{8}$/)

  @Matches(/^\d{8}$/)
  endDate: string

  @IsOptional()
  @IsString()
  tsCode?: string

}
```

## 四、Feature B：价格预警规则管理

### 4.1 接口设计

| 方法 | 路径 | 说明 |
|---|---|---|
| `POST` | `/alert/price-rules` | 创建预警规则 |
| `GET` | `/alert/price-rules` | 查询我的规则列表 |
| `PATCH` | `/alert/price-rules/:id` | 更新规则 |
| `DELETE` | `/alert/price-rules/:id` | 删除规则（软删除，改 status = DELETED） |
| `POST` | `/alert/price-rules/scan` | 管理端：手动触发扫描（RolesGuard, ADMIN） |

### 4.2 预警规则类型

```typescript
enum PriceAlertRuleType {
  PCT_CHANGE_UP   = 'PCT_CHANGE_UP',   // 当日涨幅 >= threshold%
  PCT_CHANGE_DOWN = 'PCT_CHANGE_DOWN', // 当日跌幅 <= -threshold%（threshold 填正数）
  PRICE_ABOVE     = 'PRICE_ABOVE',     // 收盘价 >= threshold
  PRICE_BELOW     = 'PRICE_BELOW',     // 收盘价 <= threshold
  LIMIT_UP        = 'LIMIT_UP',        // 涨停（close >= up_limit）
  LIMIT_DOWN      = 'LIMIT_DOWN',      // 跌停（close <= down_limit）
}

enum PriceAlertRuleStatus {
  ACTIVE  = 'ACTIVE',
  PAUSED  = 'PAUSED',
  DELETED = 'DELETED',
}
```

### 4.3 Prisma Model

```prisma
/// 价格预警规则
model PriceAlertRule {
  id         Int                  @id @default(autoincrement())
  userId     Int                  @map("user_id")
  tsCode     String               @map("ts_code") @db.VarChar(16)  // 目标股票
  stockName  String?              @map("stock_name") @db.VarChar(64)

  ruleType   PriceAlertRuleType   @map("rule_type")
  threshold  Float?               // PCT_CHANGE_UP/DOWN 或 PRICE_ABOVE/BELOW 时必填；LIMIT_UP/DOWN 为 null
  memo       String?              @db.VarChar(256)
  status     PriceAlertRuleStatus @default(ACTIVE)

  lastTriggeredAt  DateTime? @map("last_triggered_at")  // 最近一次触发时间（盘后扫描日期）
  triggerCount     Int       @default(0) @map("trigger_count")

  createdAt  DateTime @default(now()) @map("created_at")
  updatedAt  DateTime @updatedAt @map("updated_at")

  @@index([userId, status])
  @@index([tsCode, status])
  @@map("price_alert_rules")
}

enum PriceAlertRuleType {
  PCT_CHANGE_UP
  PCT_CHANGE_DOWN
  PRICE_ABOVE
  PRICE_BELOW
  LIMIT_UP
  LIMIT_DOWN

  @@map("price_alert_rule_type")
}

enum PriceAlertRuleStatus {
  ACTIVE
  PAUSED
  DELETED

  @@map("price_alert_rule_status")
}
```

**说明**：不新增触发日志表，触发记录通过 WebSocket 实时推送即可；`triggerCount` 和 `lastTriggeredAt` 足以支持用户查看历史。若后续需要完整触发历史，可单独加表。

### 4.4 盘后扫描逻辑（每日 19:00）

```
1. 从 price_alert_rules 取全部 ACTIVE 规则
2. 按 ts_code 去重，批量从 stock_daily_prices 取最新一日行情
   - 同时 JOIN stock_limit_prices 取涨跌停价（供 LIMIT_UP/DOWN 规则使用）
3. 逐规则检查：
   a. PCT_CHANGE_UP   → daily.pct_chg >= threshold
   b. PCT_CHANGE_DOWN → daily.pct_chg <= -threshold
   c. PRICE_ABOVE     → daily.close >= threshold
   d. PRICE_BELOW     → daily.close <= threshold
   e. LIMIT_UP        → daily.close >= stk_limit.up_limit
   f. LIMIT_DOWN      → daily.close <= stk_limit.down_limit
4. 命中的规则：
   - 更新 lastTriggeredAt、triggerCount++
   - 通过 EventsGateway.emitToUser(rule.userId, 'price-alert', payload) 推送
5. 日志：Logger 打印命中数量；不写 DB 触发日志（本期不做）
```

**WebSocket 推送 payload：**

```typescript
interface PriceAlertPayload {
  ruleId: number
  tsCode: string
  stockName: string | null
  ruleType: PriceAlertRuleType
  threshold: number | null
  tradeDate: string    // YYYYMMDD
  actualValue: number  // 实际涨跌幅 / 收盘价
  memo: string | null
}
```
WebSocket 事件名：`price-alert`

### 4.5 Create/Update DTO

```typescript
export class CreatePriceAlertRuleDto {
```

## 五、Feature C：异动监控引擎

异动监控**不提供对外查询 API**，也无需用户创建规则——每日盘后自动扫描全市场，将命中的异动写入一张公共表，再通过 WebSocket 向**当时在线的管理员**广播。

前端可通过 `GET /alert/anomalies` 分页查询历史异动记录。

### 5.1 Prisma Model

```prisma
/// 盘后异动监控记录
model MarketAnomaly {
  id          Int              @id @default(autoincrement())
  tradeDate   DateTime         @map("trade_date") @db.Date
  tsCode      String           @map("ts_code") @db.VarChar(16)
  stockName   String?          @map("stock_name") @db.VarChar(64)
  anomalyType MarketAnomalyType @map("anomaly_type")
  value       Float            // 触发指标的实际值
  threshold   Float            // 判定阈值
  detail      Json             @default("{}")  // 附加上下文（如近 20 日均量、大单金额等）
  scannedAt   DateTime         @default(now()) @map("scanned_at")

  @@unique([tradeDate, tsCode, anomalyType])
  @@index([tradeDate])
  @@index([anomalyType, tradeDate])
  @@map("market_anomalies")
}

enum MarketAnomalyType {
  VOLUME_SURGE        // 成交量异常放大（当日量 >= N 倍 20 日均量）
  CONSECUTIVE_LIMIT_UP  // 连板（连续 N 日涨停）
  LARGE_NET_INFLOW    // 超大单净流入异常（buy_elg_amount - sell_elg_amount / total_amount）

  @@map("market_anomaly_type")
}
```

### 5.2 异动判定规则

| 类型 | 触发条件 | 阈值（可在 service 内常量配置） | value 字段含义 |
|---|---|---|---|
| `VOLUME_SURGE` | `daily.vol >= avg20Vol * VOLUME_SURGE_MULTIPLIER` | `VOLUME_SURGE_MULTIPLIER = 3.0` | 当日量 / 20 日均量倍数 |
| `CONSECUTIVE_LIMIT_UP` | 连续 N 日 `close >= up_limit` | `CONSECUTIVE_LIMIT_MIN = 2`（连续 2 日以上） | 连板天数 |
| `LARGE_NET_INFLOW` | `(buyElgAmount - sellElgAmount) / amount >= LARGE_INFLOW_RATIO` | `LARGE_INFLOW_RATIO = 0.15`（净流入占总成交 15%） | 净流入比例 |

### 5.3 扫描逻辑（与价格预警共用同一 cron，每日 19:00）

```
1. 获取最新交易日（取 stock_daily_prices 中最近日期）
2. VOLUME_SURGE：
   - 从 stock_daily_prices 取最新日全市场数据
   - 按 ts_code 取近 21 日（最新日 + 前 20 日）计算 20 日均量
   - vol >= avg20Vol * 3.0 → 写入 market_anomalies
3. CONSECUTIVE_LIMIT_UP：
   - JOIN stock_limit_prices 对比 close vs up_limit，取近 5 日
   - 连续 2 日以上 close >= up_limit → 写入
4. LARGE_NET_INFLOW：
   - 全市场当日 stock_capital_flows
   - (buyElgAmount - sellElgAmount) / amount >= 0.15 → 写入
5. upsert（@@unique 保证幂等）
6. 统计新增条数，通过 EventsGateway.broadcastNotification 广播汇总
   WebSocket 事件名：`market-anomaly-scan-completed`
   payload：{ tradeDate, volumeSurgeCount, limitUpCount, largeInflowCount, totalNew }
```

### 5.4 查询接口

```
GET /alert/anomalies
```

**Query 参数：**

| 参数 | 类型 | 说明 |
|---|---|---|
| `tradeDate` | `string (YYYYMMDD)` | 不传则取最新交易日 |
| `type` | `MarketAnomalyType` | 不传则全部 |
| `tsCode` | `string` | 可选，按股票筛选 |
| `page` | `number` | 默认 1 |
| `pageSize` | `number` | 默认 20，最大 100 |

**响应：** 标准分页结构，data 为 `MarketAnomalyDto[]`，含 `tradeDate, tsCode, stockName, anomalyType, value, threshold, detail`。

## 附：Cron 调度说明

价格预警扫描（`PriceAlertService`）和异动扫描（`MarketAnomalyService`）均在每日 **19:00 Asia/Shanghai** 触发，与 `EventSignalService` 相同时间窗口。三者各自独立的 `@Cron` 装饰器，互不阻塞。如需调整时间，通过各自 cron 表达式配置。

管理端手动触发接口（`POST /alert/anomalies/scan` 和 `POST /alert/price-rules/scan`）供调试使用，需 ADMIN 角色。
