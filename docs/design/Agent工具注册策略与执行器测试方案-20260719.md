## 3. 业务理解

### 3.1 业务场景

- 模型请求固定 Tool 时，系统先解析冻结版本、验证参数和授权，再触达 adapter。
- Tool 超时、取消、重试或输出不可信时，执行器 fail-closed，不把伪事实交给模型。
- 每次调用和 attempt 都有脱敏审计；日志只含 trace-safe 元数据。

### 3.2 核心业务规则

| 规则编号 | 规则                                                                                     | 来源                         | 测试影响                                       |
| -------- | ---------------------------------------------------------------------------------------- | ---------------------------- | ---------------------------------------------- |
| R-001    | Registry 默认拒绝；只有构建期注册且显式 enabled 的 key/version 可执行                    | Batch 006、ADR-004           | 空配置、未知 key/version、disabled 全部拒绝    |
| R-002    | input schema 根节点必须 object 且 `additionalProperties=false`；schema 必须启动时可编译  | Tool 标准                    | 额外字段、`userId` 注入、非法 schema fail-fast |
| R-003    | userId/role/status/scope/run/workflow/budget 只来自 server context                       | Tool 标准                    | 模型 input 不能提权或跨租户                    |
| R-004    | MVP 只允许 READ；WRITE/DESTRUCTIVE definition 启动失败                                   | Batch 006                    | 恶意注册不能进入运行态                         |
| R-005    | 角色顺序 USER < ADMIN < SUPER_ADMIN；用户必须 ACTIVE；scope 与 workflow allowlist 取交集 | Policy 设计                  | 全部拒绝发生在 adapter 前                      |
| R-006    | 先写审计 start，再执行；审计 start 失败不触达 adapter；终态审计失败不返回成功            | Batch 003/006                | audit fail-closed                              |
| R-007    | 只有 idempotent READ 且 error.retryable 才自动重试；每次 attempt 单独记录                | Tool 标准                    | 权限/参数/输出错误零重试                       |
| R-008    | parent cancel 优先于 timeout；timeout/cancel 后迟到结果不能写 success                    | 状态模型                     | 终态唯一，late result 被忽略                   |
| R-009    | output 必须满足 schema、ToolResult provenance、不超 maxRows/bytes                        | Tool 标准                    | 非法、过大、缺来源结果失败                     |
| R-010    | per-run call count 与并发上限原子检查，达到边界后拒绝                                    | Batch 006、Orchestrator 约束 | 并发不能绕过 quota                             |
| R-011    | Registry snapshot 与 provider schema 序列化按 key/version 稳定排序并冻结                 | Workflow 版本治理            | 同一 snapshot hash/JSON 稳定                   |
| R-012    | canonical 15 key 只声明能力；未实现 key 不得出现在 provider schema                       | Batch 006                    | app 默认无可调用 Tool                          |

### 3.3 状态变化与不变量

| 类型           | 说明                                                                                     |
| -------------- | ---------------------------------------------------------------------------------------- |
| 数据写入       | ToolCall 从 start 到 success/failure/rejected；attempt 与同一逻辑 call 绑定              |
| 缓存/队列/通知 | 本批不启用缓存/队列/通知；避免无数据版本时产生跨权限缓存                                 |
| 权限边界       | 认证 context 不进入 JSON Schema；管理员不自动绕过资源 owner                              |
| 时间规则       | timeout 取 definition、系统默认和 Run 剩余 deadline 的最小值                             |
| 数据不变量     | key/version 唯一；version 正整数；attempt/maxRows/timeout/bytes 有界；结果 JSON 可序列化 |

## 6. 风险分析

| 风险                          | 业务影响                   | 测试策略                            | 优先级 |
| ----------------------------- | -------------------------- | ----------------------------------- | ------ |
| 模型伪造 userId/role          | 跨租户读取持仓和回测       | strict schema + server context 注入 | P0     |
| disabled/未知 Tool 被调用     | 绕过发布流程或触达危险依赖 | 默认 deny + frozen snapshot         | P0     |
| timeout 后迟到 success        | 已取消任务出现伪成功事实   | AbortSignal + settled gate          | P0     |
| retry 非幂等调用              | 重复副作用/费用            | READ+idempotent+retryable 三重门禁  | P0     |
| output schema/provenance 缺失 | 模型使用错误金融事实       | 输出校验后才审计 success            | P0     |
| 并发绕过 quota                | 单 Run 放大 DB/供应商压力  | per-run bulkhead 原子计数           | P1     |

## 8. 性能与压力测试设计

| 项目       | 设计                                                         |
| ---------- | ------------------------------------------------------------ |
| 环境       | macOS arm64、单 Jest worker、无 DB/Redis/网络                |

| 指标       | p50/p95/p99、throughput、错误率、最大 in-flight、heap 前后差 |
| 阈值或基线 | 正确性：错误率 0、无越权/超限；性能仅记录本机基线            |
