# 行业字典统一映射 — 后端设计（方案 D）

> 申万 L1 主字典 + 东财行业板块映射服务  
> 目标：让前端跨 Tab 联动不再靠行业名称模糊匹配，而是拿到稳定的 `dcTsCode`，精确跳转到行业轮动模块。

---

## 一、背景与问题分析

### 当前两套行业字典的差异

基于本地真实数据验证（2026-04-27）：

| 数据源                     | 来源表                          | 行业数量             | 说明                                    |
| -------------------------- | ------------------------------- | -------------------- | --------------------------------------- |
| `/api/heatmap/data`        | `stock_basic_profiles.industry` | **110 个**行业字符串 | 非标准分类，来自 Tushare 股票基本档案   |
| `/api/industry-rotation/*` | `sector_capital_flows`（东财）  | **496 个**行业板块   | `content_type = '行业'`，含多层级多粒度 |
| 两者直接名称精确匹配       | —                               | **35 个**            | 缺口 75 个，不适合作为跨 Tab 联动主键   |

名称差异典型案例（无法靠字符串归一化解决）：

```
stock_basic.industry  vs.  sector_capital_flows.name
食品                      食品饮料
食品饮料                   食品饮料
食品加工                   食品饮料
白酒Ⅱ                    白酒
饮料乳品                   乳制品
```

### 已有更好的标准数据

本地库已同步 Tushare 申万行业体系：

- `sw_industry_classification`：申万行业分类（`index_classify`）
- `sw_industry_members`：申万行业成分（`index_member_all`）

**真实库验证结果**：

| 指标                           | 数据                    |
| ------------------------------ | ----------------------- |
| 申万 L1 行业数量               | 31 个（SW2021）         |
| 申万 L1 与东财行业板块精确匹配 | **31/31（100%）**       |
| 上市股票申万 L1 覆盖率         | **5491/5510（99.65%）** |

### 结论

**最优解不是把 `stock_basic.industry` 当申万行业，而是用真正的申万 L1 表作为主字典。**

## 二、总体架构

### 当前架构（有缺陷）

```
/api/heatmap/data  →  stock_basic_profiles.industry（110 个字符串）
                              ↓
                      名称模糊匹配（不稳定）
                              ↓
/api/industry-rotation/*  →  sector_capital_flows（496 个东财板块）
```

### 目标架构（方案 D）

```
sw_industry_classification（申万 L1 31 个）
                ↓
     IndustryDictMappingService（统一行业映射）
                ↓
sector_capital_flows（东财行业板块）

/api/heatmap/data（industry_source=sw_l1）  ─── 使用统一映射 ──→
/api/industry-rotation/*                    ←── dcTsCode 精确联动

前端：拿到 dcTsCode 精确跳转，不再模糊匹配
```

## 六、数据来源与映射规则

### 主字典：申万 L1

```sql
-- 来源
SELECT index_code, industry_name, src
FROM sw_industry_classification
WHERE level = 'L1'
```

| 字段            | 说明                             |
| --------------- | -------------------------------- |
| `index_code`    | 申万行业指数代码，如 `801120.SI` |
| `industry_name` | 申万行业名，如 `食品饮料`        |
| `src`           | 版本标识，如 `SW2021`            |

### 个股归属：申万成分

```sql
-- 来源（取当前在册成员）
SELECT DISTINCT ts_code, l1_code, l1_name
FROM sw_industry_members
WHERE is_new = 'Y'
```

### 目标字典：东财行业板块

```sql
-- 来源（取最新交易日）
SELECT DISTINCT ts_code,
  regexp_replace(ts_code, '\.DC$', '') AS board_code,
  name
FROM sector_capital_flows
WHERE content_type = '行业'
  AND trade_date = (SELECT MAX(trade_date) FROM sector_capital_flows WHERE content_type = '行业')
```

### 映射策略（第一版）

1. 取申万 L1 行业（31 个）
2. 取最新交易日东财行业板块
3. 按 `sw.industry_name = dc.name` **精确匹配**
4. 命中 → `matchType = exact`，`confidence = 1`
5. 未命中 → `dcTsCode = null`，`matchType = none`

当前真实库 31/31 全部精确匹配，第一版不需要模糊匹配逻辑。

## 七、SQL 设计

### 字典映射核心 SQL

```sql
WITH latest_dc AS (
  SELECT MAX(trade_date) AS trade_date
  FROM sector_capital_flows
  WHERE content_type = '行业'
),
dc AS (
  SELECT DISTINCT
    ts_code,
    regexp_replace(ts_code, '\.DC$', '') AS board_code,
    name
  FROM sector_capital_flows
  WHERE content_type = '行业'
    AND trade_date = (SELECT trade_date FROM latest_dc)
    AND name IS NOT NULL
),
sw AS (
  SELECT index_code, industry_name, src
  FROM sw_industry_classification
  WHERE level = 'L1'
)
SELECT
  sw.index_code         AS sw_code,
  sw.industry_name      AS sw_name,
  dc.ts_code            AS dc_ts_code,
  dc.board_code         AS dc_board_code,
  dc.name               AS dc_name,
  CASE
    WHEN dc.ts_code IS NOT NULL THEN 'exact'
    ELSE 'none'
  END                   AS match_type,
  CASE
    WHEN dc.ts_code IS NOT NULL THEN 1
    ELSE 0
  END                   AS confidence,
  sw.src                AS version,
  (SELECT trade_date FROM latest_dc) AS trade_date
FROM sw
LEFT JOIN dc ON dc.name = sw.industry_name
ORDER BY sw.index_code;
```

### 热力图 `sw_l1` SQL 思路

```sql
WITH current_sw AS (
  SELECT DISTINCT ON (ts_code)
    ts_code, l1_code, l1_name
  FROM sw_industry_members
  WHERE is_new = 'Y'
  ORDER BY ts_code, in_date DESC NULLS LAST
),
latest_dc AS (
  SELECT MAX(trade_date) AS trade_date
  FROM sector_capital_flows
  WHERE content_type = '行业'
),
dc AS (
  SELECT DISTINCT
    ts_code,
    regexp_replace(ts_code, '\.DC$', '') AS board_code,
    name
  FROM sector_capital_flows
  WHERE content_type = '行业'
    AND trade_date = (SELECT trade_date FROM latest_dc)
)
SELECT
  sb.ts_code                              AS "tsCode",
  sb.name                                 AS "name",
  COALESCE(sw.l1_name, '未分类')          AS "groupName",
  sw.l1_code                              AS "swCode",
  sw.l1_name                              AS "swName",
  dc.ts_code                              AS "dcTsCode",
  dc.board_code                           AS "dcBoardCode",
  dc.name                                 AS "dcName",
  d.pct_chg                               AS "pctChg",
  db.total_mv                             AS "totalMv",
  d.amount                                AS "amount"
FROM stock_basic_profiles sb
JOIN stock_daily_prices d
  ON sb.ts_code = d.ts_code
 AND d.trade_date = $1::date
LEFT JOIN stock_daily_valuation_metrics db
  ON sb.ts_code = db.ts_code
 AND db.trade_date = $1::date
LEFT JOIN current_sw sw
  ON sw.ts_code = sb.ts_code
LEFT JOIN dc
  ON dc.name = sw.l1_name
WHERE sb.list_status = 'L'
ORDER BY COALESCE(sw.l1_name, '未分类'), db.total_mv DESC NULLS LAST;
```

## 八、缓存策略

### 字典映射接口缓存

```text
Key：industry:dict-mapping:sw_l1:dc_industry:{latestDcTradeDate}:{version}
TTL：24h
```

原因：申万行业分类变化极低频，东财行业板块 L1 名称映射几乎不变。

### 热力图缓存 Key 需加维度

```text
heatmap:data:{tradeDate}:{groupBy}:{industrySource}:{includeMapping}:{limit}
```

防止不同参数组合串缓存。

## 十、测试方案

### Service 单测（`industry-dict.service.spec.ts`）

1. 31 个申万 L1 全部精确匹配时，`coverage.matched = 31`，`matchRate = 1`
2. 某个申万行业找不到东财板块时，返回 `dcTsCode = null`，`matchType = 'none'`
3. `includeUnmatched=false` 时，过滤 `matchType = 'none'` 的项
4. `dcBoardCode` 正确去掉 `.DC` 后缀
5. `coverage.matchRate` 小数计算正确（`matched / total`）
6. 上市股票覆盖率统计使用 `is_new = 'Y'` 条件

### Controller 测试（`industry.controller.spec.ts`）

1. `POST /industry/dict-mapping` 返回 201
2. 返回体被 `TransformInterceptor` 包装后 `code = 0`
3. 请求体为空时走默认参数（`source=sw_l1`，`target=dc_industry`）
4. 非法 `source` 或 `target` 返回 400

### Heatmap 测试

1. 不传 `industry_source` 时，旧逻辑不变，返回字段不含 `swCode`
2. `industry_source=sw_l1` 时，item 含 `swCode/swName/dcTsCode/dcName`
3. 未映射股票进入 `groupName = '未分类'`，映射字段为 null
4. `include_mapping=false` 时，不返回映射字段（或返回 null，行为需固定）

### Industry Rotation Detail 测试

1. 只传 `industry`，旧逻辑可用
2. 只传 `tsCode`，新逻辑可用
3. 同时传 `tsCode` 和 `industry`，优先 `tsCode`
4. 两者都不传，返回 400

### 前端开发 Prompt：行业分析跨 Tab 联动改造（方案 D）

你现在要改造"行业分析"模块的跨 Tab 联动逻辑。后端采用"方案 D：申万 L1 主字典 + 东财行业板块映射服务"。

#### 背景

当前"全景热力图"和"行业轮动"使用了不同的行业字典：

- `/api/heatmap/data` 旧逻辑来自 `stock_basic_profiles.industry`，是个股档案里的行业字符串，真实有 **110 个行业字符串**。
- `/api/industry-rotation/*` 来自 `sector_capital_flows`，是东财行业板块，使用 `BKxxxx.DC` 格式代码，最新有 **496 个板块**。
- 两者直接名称完全匹配只有 **35 个**，不能靠模糊匹配做稳定联动。

后端已提供统一行业映射服务：

- 主字典：申万 L1 行业，31 个（SW2021），来自 Tushare `index_classify`。
- 目标字典：东财行业板块，来自 Tushare `moneyflow_ind_dc`。
- 后端验证结果：申万 L1 与东财行业板块 **31/31 精确匹配**，上市股票申万 L1 覆盖率 **99.65%**。
- 前端跨 Tab 联动应优先使用后端返回的 `dcTsCode`，不要再自己做模糊匹配。

   - `swName → mapping`
   - `dcTsCode → mapping`
4. 接口失败时，不影响页面主体展示；禁用跨 Tab 精确跳转，显示轻提示即可。

新增 hook 建议：

```ts
useIndustryDictMapping()
```

新增工具函数建议：

```ts
// src/utils/industryMapping.ts
buildIndustryMappingIndexes(items: IndustryDictMappingItem[])
resolveDcTsCodeFromHeatmapItem(item: HeatmapItem, indexes)
formatIndustryDictStatus(coverage: IndustryDictMappingCoverage)
```

#### 接口 2：热力图数据增强

```http
POST /api/heatmap/data
```

新请求体：

```json
{
  "group_by": "industry",
  "industry_source": "sw_l1",
  "include_mapping": true,
  "trade_date": "20260427"
}
```

兼容说明：

- 旧请求不传 `industry_source`，后端保持原逻辑，不影响已有页面。
- 新行业分析页面请传 `industry_source: 'sw_l1'` 和 `include_mapping: true`。

热力图 item 类型扩展（新字段全部 optional）：

```ts
export interface HeatmapItem {
  tsCode: string
  name: string | null
  groupName: string | null
  industry: string | null
  pctChg: number | null
  totalMv: number | null
  amount: number | null

  // 新增，industry_source=sw_l1 且 include_mapping=true 时存在
  swCode?: string | null
  swName?: string | null
  dcTsCode?: string | null
  dcBoardCode?: string | null
  dcName?: string | null
}
```

**前端分组/展示逻辑**：

1. `industry_source=sw_l1` 时，用 `swCode` 或 `swName` 作为行业 group key。
2. 展示名称优先级：`swName` → `groupName` → `industry` → `'未分类'`
3. 点击行业时，`dcTsCode` 获取优先级：
   - item 自带 `dcTsCode`
   - 用 `swCode` 查字典索引
   - 用 `swName/groupName/industry` 查字典索引
4. 如果最终没有 `dcTsCode`，**不要静默失败**，展示文案：
   - `该行业在轮动数据中暂未找到对应板块（行业字典差异）`

#### 跨 Tab 联动完整逻辑

```ts
function handleHeatmapIndustryClick(itemOrGroup: HeatmapItem | HeatmapGroup) {
  const dcTsCode =
    itemOrGroup.dcTsCode ??
    findMappingBySwCode(itemOrGroup.swCode)?.dcTsCode ??
    findMappingBySwName(itemOrGroup.swName ?? itemOrGroup.groupName ?? itemOrGroup.industry)?.dcTsCode

  if (!dcTsCode) {
    showSubtitleHint('该行业在轮动数据中暂未找到对应板块（行业字典差异）')
    return
  }

  navigateToIndustryRotation({
    swCode: itemOrGroup.swCode,
    swName: itemOrGroup.swName ?? itemOrGroup.groupName ?? itemOrGroup.industry,
    dcTsCode,
    dcName: itemOrGroup.dcName ?? findMappingByDcTsCode(dcTsCode)?.dcName,
  })
}
```

行业轮动页面收到跳转状态后：

1. 自动切换到行业轮动 Tab。
2. 高亮对应行业。
3. 请求详情：

```ts
fetchIndustryRotationDetail({
  tsCode: dcTsCode,
  industry: dcName ?? swName,
  days: selectedDays,
})
```

4. 如果详情接口返回空，展示：`该行业暂无轮动详情数据`

## 十三、推荐落地顺序

| 阶段 | 后端                                             | 前端                                   | 风险                 |
| ---- | ------------------------------------------------ | -------------------------------------- | -------------------- |
| 1    | 实现 `POST /api/industry/dict-mapping`           | 接入字典接口，改用 `dcTsCode` 精确跳转 | 最低，纯新增         |
| 2    | `/api/heatmap/data` 支持 `industry_source=sw_l1` | 热力图行业口径切换为 `sw_l1`           | 中（需切换分组依据） |
| 3    | `/api/industry-rotation/detail` 支持 `tsCode`    | 跳转详情优先传 `tsCode`                | 低，兼容旧逻辑       |
| 4    | 可选：人工 override 表                           | —                                      | 按需                 |

**原则：先加新能力，不破坏旧接口；前端逐步切换，随时可降级。**
