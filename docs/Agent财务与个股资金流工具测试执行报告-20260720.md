## 1. 本轮目标

- 验证历史财务查询只读取请求时点已实际公告的数据，不产生未来函数。
- 验证利润表/现金流累计转单季、资产负债表时点口径、指标 allowlist、单位和 null 语义。
- 验证 Tushare 官方 `net_mf_amount` 不被分单买卖差额错误替代。
- 验证最大结果、真实查询计划、并发负载、跨批次回归和新容器运行状态。

## 2. 业务理解摘要

| 项目     | 结论                                                                                        |
| -------- | ------------------------------------------------------------------------------------------- |
| 业务场景 | Agent 为盈利质量、偿债能力、现金流与资金行为研究读取可追溯事实                              |
| 核心规则 | `fAnnDate ?? annDate` 作为实际可得日；修订稳定选版；累计流量按同年上一季度相减；null 不转 0 |
| 关键风险 | 公告穿越、随机修订、累计误当单季、资产余额相减、任意指标注入、官方净流入被错误重算          |
| 边界     | 本批无 REST、migration、同步或补数；默认 `AGENT_TOOLS_ENABLED` 仍为空                       |

## 6. 发现的问题与约束

本轮未发现阻塞代码缺陷。实库与模型存在以下已处理约束：

| ID           | 严重度 | 内容                                         | 业务影响                                      | 处理结果   |
| ------------ | ------ | -------------------------------------------- | --------------------------------------------- | ---------- |
| FMT-RISK-001 | S1     | 三大报表同报告期存在 `updateFlag=0/1` 多版本 | 若直接按 endDate 取一行，结果可能随机漂移     | 已缓解     |
| FMT-RISK-002 | S1     | `FinaIndicator` 主键只保留每报告期一行       | 已被覆盖的历史修订无法完整 point-in-time 恢复 | 已显式告警 |
| FMT-RISK-003 | S2     | 财务公告只保存日历日，无盘中发布时间         | 无法宣称盘中级 point-in-time 精度             | 已显式告警 |
| FMT-RISK-004 | S2     | 开发库 Moneyflow 当前实际覆盖约 60 个交易日  | Tool 上限 250 日不等于当前一定有 250 日数据   | 已文档化   |

## 8. 真实数据正确性结果

- `600519.SH` 最新收入报告期为 `2026-03-31`，实际可得日 `2026-04-25`，稳定选择 `updateFlag=1`，`revisionCount=2`。
- 最新累计营业总收入 `54,702,912,385.23` 元；Q1 单季值与累计值相同。
- 2026-07-17 官方 `net_mf_amount=84,386.98` 万元；特大单买卖差额为 `-49,960.01` 万元，证明两者不能互相替代。
- 资产负债表只返回 `POINT_IN_TIME`；利润表和现金流仅对可加流量字段派生单季值。
- 指标返回 canonical key、Tushare source field 与 `CNY/CNY_PER_SHARE/PERCENT/RATIO` 单位；null 保持 null。
