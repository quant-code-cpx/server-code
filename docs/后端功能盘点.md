## 二、各模块端点明细

### 1. Auth（4 端点）

| 路由                 | 说明                                                                |
| -------------------- | ------------------------------------------------------------------- |
| `POST /auth/captcha` | 获取图片验证码（SVG + captchaId，60s TTL，一次性）                  |
| `POST /auth/login`   | 登录（验证码+账号密码），返回 AccessToken，RefreshToken 写入 Cookie |
| `POST /auth/refresh` | 刷新 AccessToken（Token 轮换 + 宽限期防 React StrictMode 双调用）   |
| `POST /auth/logout`  | 登出（AT 加黑名单 + 撤销 RT + 清除 Cookie）                         |

**服务能力**：连续失败锁定（Redis 计数）、不区分"账号不存在"与"密码错误"防枚举

## 三、异步任务 & 定时调度

### BullMQ 队列

| 队列                    | Job 类型               | 说明                               |
| ----------------------- | ---------------------- | ---------------------------------- |
| `backtesting`           | `run-backtest`         | 执行单次回测                       |

| `screener-subscription` | `batch_execute`        | 批量执行同频率的所有活跃订阅       |
| `screener-subscription` | `execute_subscription` | 执行单个订阅（选股→对比→推送告警） |

### Cron 定时任务

| 调度频率                 | 模块                          | 说明             |
| ------------------------ | ----------------------------- | ---------------- |
| 每日盘后（按 plan 注册） | Tushare Sync                  | 增量数据同步     |
| 每 5 分钟                | SyncRetryService              | 扫描失败重试队列 |
| 工作日 20:30             | ScreenerSubscriptionScheduler | 日频条件订阅     |
| 周一 20:30               | ScreenerSubscriptionScheduler | 周频条件订阅     |
