# 股票信息模块需求细化（讨论稿）

> 目的：把“股票列表 / 股票详情 / 选股器”三个想法细化成可讨论、可继续补充的设计文档。
> 说明：本文件**不涉及实现**，重点是需求拆分、Tushare 数据来源、建议定义的后端接口，以及对产品与财经认知层面的补充建议。

---

## 一、当前项目现状与结论

结合当前仓库代码，股票信息模块目前已有的基础能力主要是：

- 股票列表骨架：`stock_basic` 入库后支持按交易所 / 上市状态 / 行业筛选
- 股票详情骨架：返回基础信息 + 公司信息 + 最新日线 / 每日指标 / 复权因子
- 市场资金流：支持大盘资金流、行业/概念/地域板块资金流
- 已接入的 Tushare 接口：
  - `stock_basic`
  - `stock_company`
  - `trade_cal`
  - `daily` / `weekly` / `monthly`
  - `adj_factor`
  - `daily_basic`
  - `moneyflow`
  - `moneyflow_ind_dc`
  - `moneyflow_mkt_dc`
  - `express`

这意味着：

- **股票列表**已经有较好的起点，继续完善筛选、搜索、分页即可
- **股票详情**可以先做出“基础版”，但如果要真正支持分析一只股票，还需要补充更多财务、股东、分红、技术指标类数据
- **选股器**是最有价值也最容易复杂化的模块，它的关键难点不是页面，而是：
  - 指标口径统一
  - 数据时点一致
  - 筛选性能
  - 财经知识建模

---

## 二、总设计建议

从后端设计角度，建议把股票模块拆成三个层次：

1. **股票列表（List）**
   - 解决“找股票、筛股票、快速浏览”的问题
2. **股票详情（Detail）**
   - 解决“分析一只股票”的问题
3. **选股器（Screener）**
   - 解决“按规则筛出一批候选股票”的问题

建议不要把“股票详情”做成一个过于巨大的单接口，而是拆成：

- 一个总览接口
- 多个按领域拆分的子接口

原因：

- 图表数据、财务数据、资金流数据的更新频率与查询粒度不同
- 单接口过大后，不利于缓存、分页、性能控制、前端按需加载
- 后续你一定会继续加字段，拆分式设计更稳

## 三、想法 1：股票列表模块

### 3.1 目标理解

你的想法本质上是：

- 像经典后台管理系统一样，支持筛选、分页、列表展示
- 适合“先缩小范围，再进入详情分析”的工作流

这是非常合理的第一入口，因为大部分投研/选股流程都不是直接进单只股票，而是先看一个候选池。

### 3.2 建议细化的功能点

建议把股票列表拆成以下能力：

1. **基础筛选**
   - 交易所：SSE / SZSE / BSE
   - 上市状态：上市 / 暂停上市 / 退市
   - 行业
   - 地域
   - 市场板块（主板 / 创业板 / 科创板 / 北交所等）

2. **搜索能力**
   - 按 `ts_code`
   - 按 `symbol`
   - 按股票简称 `name`
   - 按拼音缩写 / 全称（如后续需要）

3. **列表展示字段**
   - 股票代码 / 名称
   - 所属行业 / 地域 / 市场
   - 最新价
   - 当日涨跌幅
   - 总市值 / 流通市值
   - PE(TTM) / PB / 股息率
   - 最近 60 日资金净流入（可选）
   - 上市日期

4. **排序能力**
   - 按市值
   - 按涨跌幅
   - 按换手率
   - 按成交额
   - 按 PE / PB / 股息率
   - 按最近 N 日资金净流入

5. **分页能力**
   - 标准分页：`page` + `pageSize`
   - 返回 `total`
   - 后续如果数据量非常大，可考虑游标分页，但目前标准分页足够

### 3.5 点评：你需要注意的点 / 需要学习的点 / 当前遗漏点

#### 需要注意的点

- 股票列表不要只停留在“基础资料表”，否则很快会发现它不够用。
- 真正有分析价值的列表，一定要把“静态信息 + 最新行情 + 最新估值 + 近期资金流”融合起来。
- 排序字段不要一开始放太多，否则 SQL 会很重，建议先做高频字段。

#### 需要学习的点

- 了解常见行情列表页的“字段优先级”，比如价格、涨跌幅、换手率、市值、PE(TTM)、股息率为什么最常见。
- 学会区分：
  - 静态字段（公司名称、行业）
  - 日频字段（价格、PE、换手率）
  - 区间聚合字段（近 20 日资金净流入）

#### 你可能遗漏的点

- **关键词搜索**：这是日常使用频率极高的能力
- **排序能力**：列表如果不能排序，实用性会差很多
- **自选股 / 观察池**：虽然不是现在必须做，但将来很自然会加
- **导出能力**：筛选结果导出 Excel / CSV 在研究场景很常见

## 四、想法 2：股票详情模块

### 4.1 目标理解

你的想法非常正确：股票详情不应该只是“基础资料 + 一个日线图”。

真正能支持分析一只股票的详情页，至少应该覆盖：

- 公司是谁
- 价格怎么走
- 当前估值如何
- 资金最近在不在流入
- 财务质量好不好
- 成长性如何
- 分红如何
- 有没有重要风险信号

### 4.2 建议拆分的详情子模块

建议把详情页拆成下面几个区域：

1. **总览卡片**
   - 股票简称 / 代码 / 所属行业 / 市场 / 地域
   - 最新价 / 涨跌幅 / 成交额 / 换手率
   - 总市值 / 流通市值
   - PE(TTM) / PB / 股息率(TTM)
   - 最近财报期 / 最近公告期

2. **K 线图与图表数据**
   - 周期：日 / 周 / 月
   - 复权方式：不复权 / 前复权 / 后复权
   - 成交量柱状图
   - 常用技术指标：MA、MACD、KDJ、RSI、BOLL

3. **关键指标区**
   - 估值指标：PE、PB、PS、股息率
   - 交易指标：换手率、量比、涨跌停状态
   - 资金指标：最近 5/20/60 日净流入

4. **资金流向区**
   - 近 60 日主力/超大单/大单/中单/小单资金流入流出趋势
   - 最近 1 日、5 日、20 日、60 日净流入汇总

5. **财务分析区**
   - 营收、净利润、毛利率、净利率、ROE、ROA
   - 资产负债率、经营现金流、自由现金流
   - 同比增速、环比增速、TTM 指标

6. **分红与股东区**
   - 历史分红次数、分红率、股息率
   - 前十大股东 / 前十大流通股东
   - 股权质押情况（如后续补）

7. **风险与事件区**
   - 业绩预告 / 业绩快报
   - 审计意见
   - 质押、商誉、减值等风险提示（后续扩展）

### 4.5 点评：你需要注意的点 / 需要学习的点 / 当前遗漏点

#### 需要注意的点

- 详情页最大的坑是“想一次把所有数据都塞进去”，这会导致接口又慢又重。
- K 线、资金流、财务数据最好拆接口，前端按需加载。
- “财务数据”不要只看营收和净利润，必须把盈利质量和现金流放进去，否则容易看错公司。

#### 需要学习的点

- 学习估值体系：PE、PB、PS、EV/EBITDA（如果以后扩展）分别适合哪些行业。
- 学习盈利质量：净利润高不等于现金流好，ROE 高不一定安全。
- 学习财报视角：利润表、资产负债表、现金流量表要结合看。
- 学习技术指标的适用边界：技术指标更像辅助观察工具，不是万能预测器。

#### 你可能遗漏的点

- **股东结构**：很多中长期分析会看前十大股东变化
- **分红与回购**：价值风格里非常重要
- **业绩预告 / 审计意见**：这是提前识别财务风险的重要线索
- **风险标签**：如高质押、高商誉、高负债、连续经营现金流为负
- **同业对比**：一只股票单看自身还不够，后面很自然会需要行业横向对比

## 五、想法 3：选股器（Stock Screener）

### 5.1 目标理解

这是三个想法里最有“研究平台味道”的模块。

你的方向已经很像一个合格的量化研究选股器了，核心维度包括：

- 板块
- 行情指标
- 估值指标
- 分红指标
- 技术指标
- 资金流向
- 盈利能力
- 成长能力
- 现金流

这已经不是简单的列表筛选，而是在构建一个“多因子条件引擎”。

### 5.2 建议先分层，而不是一次做全

建议按三层来设计：

1. **基础筛选层**
   - 行业 / 市场 / 地域 / 上市状态 / 上市年限

2. **交易与估值筛选层**
   - 最新价、市值、换手率、涨跌幅、成交额
   - PE / PB / PS / 股息率

3. **研究型筛选层**
   - 技术指标
   - 资金流向
   - 盈利能力
   - 成长能力
   - 现金流质量
   - 分红与股东结构

这样可以避免一开始就把所有条件搅成一个不可维护的大筛选器。

### 5.3 建议支持的筛选维度

#### A. 基础维度

- 交易所
- 市场板块
- 行业
- 地域
- 上市状态
- 上市天数 / 上市年限

#### B. 行情维度

- 最新价区间
- 涨跌幅区间
- 成交额区间
- 换手率区间
- 振幅区间（后续可本地计算）

#### C. 估值维度

- PE / PE(TTM)
- PB
- PS / PS(TTM)
- 股息率 / 股息率(TTM)
- 总市值 / 流通市值

#### D. 技术维度

- MA 多头排列 / 空头排列
- MACD 金叉 / 死叉
- RSI 超买 / 超卖
- KDJ 低位金叉
- 布林带突破

#### E. 资金流维度

- 最近 1/5/20/60 日净流入 > 某阈值
- 主力净流入占比 > 某阈值
- 连续 N 日净流入

#### F. 财务维度

- ROE > 某阈值
- 毛利率 > 某阈值
- 净利率 > 某阈值
- 资产负债率 < 某阈值
- 经营现金流净额 > 0
- 经营现金流 / 净利润 > 某阈值

#### G. 成长维度

- 营收同比增长率 > 某阈值
- 净利润同比增长率 > 某阈值
- 扣非净利润同比 > 某阈值
- 连续 N 期增长

#### H. 分红维度

- 最近 N 年连续分红
- 股息率(TTM) > 某阈值
- 分红率 > 某阈值

### 5.6 点评：你需要注意的点 / 需要学习的点 / 当前遗漏点

#### 需要注意的点

- 选股器的核心不是“筛选条件越多越好”，而是**条件口径要统一**。
- 如果一个条件用最新日频数据、另一个条件用上季度财报、再一个条件用过去 60 日累计值，前端必须明确告诉用户“这些指标对应的时间口径”。
- 选股器很容易变成性能热点，后期一定要考虑汇总表、宽表、缓存、预计算。

#### 需要学习的点

- 学习“因子”与“筛选条件”的区别：
  - 筛选条件是布尔过滤
  - 因子可以是排序、打分、组合权重
- 学习常见选股风格：
  - 低估值
  - 高股息
  - 高成长
  - 质量因子
  - 动量因子
  - 资金流趋势
- 学习财务指标的时点陷阱：一个季度报出来前，不能拿未来已知数据做过去时点筛选

#### 你可能遗漏的点

- **时间口径选择**：用“最新一期”还是“指定交易日可见值”
- **筛选结果解释能力**：用户不仅想知道筛出了谁，还想知道“为什么筛出来”
- **保存条件模板**：实用性非常高
- **多条件分组**：例如 `(高股息 OR 低估值) AND 资金净流入为正`
- **排序 / 打分模式**：不是只有“筛掉不符合的”，还可以“给符合的打分排名”

## 八、最后的总体点评

你的三个想法方向是对的，而且顺序也基本合理：

- 列表：入口
- 详情：分析单只股票
- 选股器：批量发现机会

如果再给一句更直接的建议，那就是：

> 你现在最需要补的，不是页面花样，而是“财务与研究数据底座”。

因为：

- 没有足够的数据底座，详情页会停留在“看图说话”
- 没有统一口径的数据底座，选股器很容易筛出“看起来合理但其实有偏差”的结果

所以后续学习与建设的重点，建议放在：

1. 财务报表理解
2. 常用估值指标适用边界
3. 因子与筛选条件的建模方式
4. 数据时点一致性
5. 选股条件的可解释性
