# 高级数据接入 — 后端设计

> **来源**：`docs/待办清单.md` P3「高级数据接入」3 项  
> **范围**：ETF/LOF 净值、宏观经济指标、期权数据  
> **日期**：2026-04-11

---

## 一、总览

本次接入 3 类 Tushare 数据（共 8 个 API），引入 2 个新的同步分类（`fund`、`macro`）和 1 个扩展分类（`alternative` 或新建 `option`）。

| 分类     | Tushare 接口 | 最低积分 | 说明                          |
| -------- | ------------ | -------- | ----------------------------- |
| **基金** | `fund_basic` | 2000     | 公募基金列表（含 ETF/LOF）    |
| **基金** | `fund_nav`   | 2000     | 公募基金净值                  |
| **基金** | `fund_daily` | 5000     | ETF 日线行情                  |
| **宏观** | `cn_cpi`     | 600      | 居民消费价格指数（CPI）       |
| **宏观** | `cn_ppi`     | 600      | 工业生产者出厂价格指数（PPI） |
| **宏观** | `cn_gdp`     | 600      | 国内生产总值（GDP）           |
| **宏观** | `shibor`     | 120      | 上海银行间同业拆放利率        |
| **期权** | `opt_basic`  | 5000     | 期权合约信息                  |
| **期权** | `opt_daily`  | 2000     | 期权日线行情                  |

> ⚠️ `fund_daily` 和 `opt_basic` 要求 **5000 积分**，请确认账户是否满足。

## 二、架构设计

### 2.1 模块扩展策略

沿用现有 plan-driven 同步架构，新增 **3 个同步服务** + **3 个 API 服务**，注册到 `TushareSyncRegistryService`：

```
src/tushare/
├── api/
│   ├── fund-api.service.ts          ← 新增：fund_basic / fund_nav / fund_daily
│   ├── macro-api.service.ts         ← 新增：cn_cpi / cn_ppi / cn_gdp / shibor
│   └── option-api.service.ts        ← 新增：opt_basic / opt_daily
├── sync/
│   ├── fund-sync.service.ts         ← 新增：FundSyncService.getSyncPlans()
│   ├── macro-sync.service.ts        ← 新增：MacroSyncService.getSyncPlans()
│   └── option-sync.service.ts       ← 新增：OptionSyncService.getSyncPlans()
├── tushare-sync.mapper.ts           ← 追加 mapper 函数
└── tushare.module.ts                ← 注册新 service
```

```typescript
constructor(
  // ...existing services
  private readonly fundSync: FundSyncService,
  private readonly macroSync: MacroSyncService,
  private readonly optionSync: OptionSyncService,
) {
  this.plans = this.collectPlans()

    ...this.macroSync.getSyncPlans(),
```

```typescript
@Injectable()
export class FundApiService {
  constructor(private readonly client: TushareClient) {}

  /** 获取基金列表 */
  getFundBasic(market: 'E' | 'O') {
    return this.client.call({
      api_name: TushareApiName.FUND_BASIC,
      params: { market },
    })
  }

  /** 按基金代码获取净值 */
  getFundNavByTsCode(tsCode: string, startDate?: string, endDate?: string) {
    return this.client.call({
      api_name: TushareApiName.FUND_NAV,
      params: { ts_code: tsCode, start_date: startDate, end_date: endDate },
      fields: FUND_NAV_FIELDS,
    })
  }

  /** 按交易日获取 ETF 日线行情 */
  getFundDailyByTradeDate(tradeDate: string) {
    return this.client.call({
      api_name: TushareApiName.FUND_DAILY,
      params: { trade_date: tradeDate },
      fields: FUND_DAILY_FIELDS,
    })
  }
}
```

### 3.4 同步策略

| 任务         | 策略                       | 说明                                                                                   |
| ------------ | -------------------------- | -------------------------------------------------------------------------------------- |
| `FUND_BASIC` | `replaceAllRows` 全量替换  | 数据量小（~15000），场内 + 场外各拉一次                                                |
| `FUND_NAV`   | 按 `ts_code` 逐只增量      | 获取 fund_basic 中**场内 ETF/LOF** 列表，按代码逐只拉取净值；增量从最新 `navDate` 继续 |
| `FUND_DAILY` | `syncByTradeDate` 按日增量 | 同 `DAILY`，按交易日拉取全市场 ETF 行情                                                |

> **注意**：`fund_nav` 一次只能按 `ts_code` 查询，ETF/LOF 约 1000 只，全量回补需控制频率。建议限制到场内基金（`market='E'`），场外基金规模太大可作为后续扩展。

```typescript
  /** 按交易所获取期权合约信息 */
  getOptBasic(exchange: string) {
    return this.client.call({
      api_name: TushareApiName.OPT_BASIC,
      params: { exchange },
      fields: OPT_BASIC_FIELDS,
    })
  }

  /** 按交易日获取期权日线行情 */
  getOptDailyByTradeDate(tradeDate: string) {
    return this.client.call({
      api_name: TushareApiName.OPT_DAILY,
      params: { trade_date: tradeDate },
      fields: OPT_DAILY_FIELDS,
    })
  }
}
```

### 5.3 同步策略

| 任务        | 策略                          | 说明                                                 |
| ----------- | ----------------------------- | ---------------------------------------------------- |
| `OPT_BASIC` | 分交易所逐一 `replaceAllRows` | SSE / SZSE / CFFEX / DCE / SHFE / CZCE 共 6 个交易所 |
| `OPT_DAILY` | `syncByTradeDate` 按日增量    | 单次最大 15000 条；按交易日拉取，与 `DAILY` 模式一致 |

## 十、数据量与频控估算

| 数据集       | 预估行数                                        | 全量回补预估请求数     | 频控风险       |
| ------------ | ----------------------------------------------- | ---------------------- | -------------- |
| `fund_basic` | ~15,000                                         | 2 次（场内+场外）      | 低             |
| `fund_nav`   | ~100 万（场内 ETF 约 1000 只 × 10 年 × 250 天） | ~1000 次（按代码逐只） | **中**，需节流 |
| `fund_daily` | ~50 万（ETF × 10 年 × 250 天）                  | ~2500 次（按交易日）   | **中**         |
| `cn_cpi`     | ~400                                            | 1 次                   | 极低           |
| `cn_ppi`     | ~300                                            | 1 次                   | 极低           |
| `cn_gdp`     | ~300                                            | 1 次                   | 极低           |
| `shibor`     | ~5000                                           | 3-5 次                 | 低             |
| `opt_basic`  | ~50,000                                         | 6 次（按交易所）       | 低             |
| `opt_daily`  | ~200 万（约 50000 合约，但活跃期短）            | ~2500 次（按交易日）   | **中**         |

> 高频控风险的数据集（fund_nav、fund_daily、opt_daily）全量回补时启用断点续传 + 节流。
