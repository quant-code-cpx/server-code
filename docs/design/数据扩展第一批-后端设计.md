# Tushare 数据扩展第一批 — 后端设计

> **范围**：`forecast`（业绩预告）、`stk_holdernumber`（股东人数）、`hk_hold`（沪深股通持股明细）、`index_dailybasic`（指数每日指标）
>
> **积分要求**：前三个 ≤ 2000，`index_dailybasic` 需 400+（官方标注 4000 但实测 400 可用）；用户当前 6000 积分全部可调
>
> **目标**：新增 4 张 Prisma 表、4 个 mapper、4 个同步计划项，注册到现有 plan-driven 同步架构

---

## 一、接口字段确认（来源：Tushare 官方文档 + 实测）

### 1.1 forecast（业绩预告，doc_id=45）

**限制**：2000 积分只能按 `ts_code` 逐个拉取历史；`forecast_vip`（5000 积分）可按 `period` 拉全市场。当前 6000 积分 → **用 `forecast_vip` 按 period 批量拉取**。

| Tushare 字段    | 类型  | 必有 | 说明                                              |
| --------------- | ----- | ---- | ------------------------------------------------- |
| ts_code         | str   | Y    | TS 股票代码                                       |
| ann_date        | str   | Y    | 公告日期                                          |
| end_date        | str   | Y    | 报告期（季末日期，如 20231231）                   |
| type            | str   | Y    | 预告类型：预增/预减/扭亏/首亏/续亏/续盈/略增/略减 |
| p_change_min    | float | N    | 净利润变动幅度下限（%）                           |
| p_change_max    | float | N    | 净利润变动幅度上限（%）                           |
| net_profit_min  | float | N    | 预告净利润下限（万元）                            |
| net_profit_max  | float | N    | 预告净利润上限（万元）                            |
| last_parent_net | float | N    | 上年同期归属母公司净利润                          |
| first_ann_date  | str   | N    | 首次公告日                                        |
| summary         | str   | N    | 业绩预告摘要                                      |
| change_reason   | str   | N    | 业绩变动原因                                      |

### 1.2 stk_holdernumber（股东人数，doc_id=166）

**限制**：600 积分可调取，单次最大 3000 行。数据不定期公布。

| Tushare 字段 | 类型 | 必有 | 说明        |
| ------------ | ---- | ---- | ----------- |
| ts_code      | str  | Y    | TS 股票代码 |
| ann_date     | str  | Y    | 公告日期    |
| end_date     | str  | Y    | 截止日期    |
| holder_num   | int  | Y    | 股东户数    |

### 1.3 hk_hold（沪深股通持股明细，doc_id=188）

**限制**：2000 积分正常使用，单次最多 3800 行。注意：交易所从 2024-08-20 起停止日度北向资金数据发布，改为季度披露。

| Tushare 字段 | 类型  | 必有 | 说明                              |
| ------------ | ----- | ---- | --------------------------------- |
| code         | str   | Y    | 原始代码                          |
| trade_date   | str   | Y    | 交易日期                          |
| ts_code      | str   | Y    | TS 代码                           |
| name         | str   | Y    | 股票名称                          |
| vol          | int   | Y    | 持股数量（股）                    |
| ratio        | float | Y    | 持股占比（%），占已发行股份百分比 |
| exchange     | str   | Y    | SH 沪股通 / SZ 深股通 / HK 港股通 |

### 1.4 index_dailybasic（大盘指数每日指标，doc_id=128）

**限制**：400 积分起，ts_code/trade_date 至少输一个，单次限量 3000 条。目前仅提供 6 个核心指数：000001.SH / 399001.SZ / 000016.SH / 000905.SH / 399005.SZ / 399006.SZ。

| Tushare 字段    | 类型  | 必有 | 说明                       |
| --------------- | ----- | ---- | -------------------------- |
| ts_code         | str   | Y    | TS 代码                    |
| trade_date      | str   | Y    | 交易日期                   |
| total_mv        | float | Y    | 当日总市值（元）           |
| float_mv        | float | Y    | 当日流通市值（元）         |
| total_share     | float | Y    | 当日总股本（股）           |
| float_share     | float | Y    | 当日流通股本（股）         |
| free_share      | float | Y    | 当日自由流通股本（股）     |
| turnover_rate   | float | Y    | 换手率                     |
| turnover_rate_f | float | Y    | 换手率（基于自由流通股本） |
| pe              | float | Y    | 市盈率                     |
| pe_ttm          | float | Y    | 市盈率 TTM                 |
| pb              | float | Y    | 市净率                     |

## 二、Prisma Schema 设计

### 2.1 `prisma/tushare_forecast.prisma`

```prisma
// 业绩预告（来自 Tushare forecast / forecast_vip 接口，doc_id=45）
model Forecast {
  tsCode        String   @map("ts_code") @db.VarChar(16)
  annDate       DateTime @map("ann_date") @db.Date
  endDate       DateTime @map("end_date") @db.Date

  type          String?  @map("type") @db.VarChar(10)
  pChangeMin    Float?   @map("p_change_min")
  pChangeMax    Float?   @map("p_change_max")
  netProfitMin  Float?   @map("net_profit_min")
  netProfitMax  Float?   @map("net_profit_max")
  lastParentNet Float?   @map("last_parent_net")
  firstAnnDate  DateTime? @map("first_ann_date") @db.Date
  summary       String?  @map("summary") @db.Text
  changeReason  String?  @map("change_reason") @db.Text

  syncedAt      DateTime @default(now()) @map("synced_at")

  @@id([tsCode, annDate, endDate])
  @@index([endDate])
  @@index([annDate])
  @@map("earnings_forecast_reports")
}
```

## 四、Mapper 函数设计

在 `src/tushare/tushare-sync.mapper.ts` 中新增 4 个函数，遵循现有 `readString/readNumber/readInt/readDate` 工具函数模式：

### 4.1 mapForecastRecord

```typescript
export function mapForecastRecord(
  r: TushareRecord,
  collector?: ValidationCollector,
): Prisma.ForecastCreateManyInput | null {
  const tsCode = readString(r, 'ts_code')
  const annDate = readDate(r, 'ann_date')
  const endDate = readDate(r, 'end_date')
  if (!tsCode || !annDate || !endDate) {
    collector?.add({ field: 'PK', message: 'ts_code/ann_date/end_date 缺失', record: r })
    return null
  }
  return {
    tsCode,
    annDate,
    endDate,
    type: readString(r, 'type'),
    pChangeMin: readNumber(r, 'p_change_min'),
    pChangeMax: readNumber(r, 'p_change_max'),
    netProfitMin: readNumber(r, 'net_profit_min'),
    netProfitMax: readNumber(r, 'net_profit_max'),
    lastParentNet: readNumber(r, 'last_parent_net'),
    firstAnnDate: readDate(r, 'first_ann_date'),
    summary: readString(r, 'summary'),
    changeReason: readString(r, 'change_reason'),
  }
}
```

## 五、同步策略设计

| 接口               | 同步策略               | 原因                                                      | 归属服务                |
| ------------------ | ---------------------- | --------------------------------------------------------- | ----------------------- |
| `forecast_vip`     | **按 period 批量**     | VIP 接口支持 `period` 参数拉全市场；遍历每个报告期        | `FinancialSyncService`  |
| `stk_holdernumber` | **按 ann_date 批量**   | 数据不定期公布，按公告日遍历；日常同步拉最近 30 天公告    | `FinancialSyncService`  |
| `hk_hold`          | **按 trade_date 批量** | 每日更新，按交易日遍历；单次最多 3800 行（SH+SZ+HK 合计） | `FactorDataSyncService` |
| `index_dailybasic` | **按 trade_date 批量** | 按交易日遍历，一次拉 6 个指数（6 行/天）；归入指数域      | `MarketSyncService`     |

### 5.1 forecast 同步逻辑

```
日常同步：
  1. 获取最近 4 个报告期（如 20261231, 20260930, 20260630, 20260331）
  2. 对每个 period 调用 forecast_vip(period=xxx)
  3. deleteMany(endDate=period) + createMany（全量替换该报告期数据）

回补同步：
  1. 从 2010 年 Q1 开始，遍历所有报告期
  2. 按上述逻辑逐期拉取替换
```

### 5.2 stk_holdernumber 同步逻辑

```
日常同步：
  1. 取最近 30 天交易日列表
  2. 对每个日期调用 stk_holdernumber(ann_date=xxx)
  3. 去重后 upsert（PK = tsCode + endDate）

回补同步：
  1. 从 2015 年开始，按月遍历 ann_date 范围
  2. 使用 start_date + end_date 参数按月拉取
```

### 5.3 hk_hold 同步逻辑

```
日常同步：
  1. 取最近 5 个交易日
  2. 对每个交易日调用 hk_hold(trade_date=xxx)（SH + SZ 分别拉取或一次拉取）
  3. deleteMany(tradeDate=xxx) + createMany

回补同步：
  1. 从 2017-03-17（深港通开通日）开始，遍历所有交易日
  2. 注意：2024-08-20 后改为季度披露，日度数据停止
```

### 5.4 index_dailybasic 同步逻辑

```
日常同步：
  1. 取最近 5 个交易日
  2. 对每个交易日调用 index_dailybasic(trade_date=xxx)
  3. upsert（PK = tsCode + tradeDate）

回补同步：
  1. 从 2004-01-02 开始，遍历所有交易日
  2. 每次返回 ≤ 6 行（6 个指数），流量压力极小
```

## 八、频控与风险评估

| 接口               | 每分钟频次（6000 积分） | 日常同步请求次数         | 回补请求次数                            | 风控策略            |
| ------------------ | ----------------------- | ------------------------ | --------------------------------------- | ------------------- |
| `forecast_vip`     | ~200                    | 4 次（4 个报告期）       | ~260 次（2010 至今约 65 年报 × 4 季度） | 无压力              |
| `stk_holdernumber` | ~100                    | ~20 次（近 30 天公告日） | ~130 次（2015 至今按月）                | 低压力              |
| `hk_hold`          | ~100                    | 5 次（近 5 交易日）      | ~1800 次（2017-2024 约 1800 个交易日）  | 中等，300ms 节流    |
| `index_dailybasic` | ~200                    | 5 次                     | ~5500 次（2004 至今）                   | 低压力，每次仅 6 行 |

总体评估：**日常同步<40 次/日，无频控风险；回补时 `hk_hold` 量最大，需 300ms 间隔节流。**
