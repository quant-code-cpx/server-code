## 1. 本轮目标

- 验证收益、波动、Sharpe、Sortino、回撤、VaR/CVaR 由固定算法计算，不由模型心算。
- 验证单股历史估值分位使用真实数据日、固定过滤/缩尾/秩定义和最小样本。
- 验证组合与回测 userId 所有权、跨租户 fail-closed、null 保真和资源上限。
- 验证历史回测偏差 warning 不可被调用方隐藏，组合 Beta 按共有交易日对齐。
- 验证真实 DB、性能、回归、编译、容器启动与 health/readiness。

## 2. 业务理解摘要

| 项目     | 结论                                                                                           |
| -------- | ---------------------------------------------------------------------------------------------- |
| 业务场景 | Agent 查询组合风险、复盘回测、计算绩效和判断单股估值位置                                       |
| 核心规则 | 算法/version/hash 固定；收益用小数；用户资源由 context 注入 userId；实际数据日不得伪装成请求日 |
| 关键风险 | 模型自定义公式、百分数放大 100 倍、重复日期、零分母、跨租户、停牌日 Beta 错配、回测偏差被隐藏  |
| 边界     | 本批无 REST、migration、新回测、Python 或历史回测偏差修复；默认 Tool allowlist 为空            |

## 6. 发现并修复的问题

| ID      | 严重度 | 问题                                                            | 处理                                                        |
| ------- | ------ | --------------------------------------------------------------- | ----------------------------------------------------------- |
| QNT-001 | S1     | 组合风险原 Beta 按数组位置配对股票和基准收益；停牌/缺失日会错位 | 改为按共有交易日内连接，并补手算 Beta 测试                  |
| QNT-002 | S1     | 风险快照使用交易日历日期；当天行情未入库时仍返回当天 `dataAsOf` | 改为查询 `DailyBasic` 真实最新日期 `<= asOf`                |
| QNT-003 | S2     | 历史组合仅有当前持仓模型，无法还原历史持仓                      | 历史 asOf 返回 `PORTFOLIO_HOLDINGS_NOT_POINT_IN_TIME`       |
| QNT-004 | S1     | 现有历史回测的 universe/公告/复权仍未完成点时验证               | 始终返回 `BACKTEST_BIAS_UNVERIFIED` 与 `reproducible=false` |
| QNT-005 | S2     | 回测归因未持久化可复现结果                                      | 请求 ATTRIBUTION 时返回 unavailable/partial，不现场重算     |

## 8. 安全与资源边界

- 四 Tool 均为 READ/idempotent；无确认、写入、删除或用户代码执行。
