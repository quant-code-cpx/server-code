# 测试待办清单

> **目标**：为本量化后端项目逐步建立可维护的自动化测试体系。
> **当前状态**：45+ 服务，14 个 Controller，47 个测试文件（664 个用例），零 E2E 测试。
> **原则**：先搭基础设施，再按业务价值从高到低逐层覆盖；每个阶段产出可独立运行的测试。

---

## 阶段概览

| 阶段   | 主题                        | 优先级  | 预计文件数 | 前置依赖 |
| ------ | --------------------------- | ------- | ---------- | -------- |
| **P0** | 测试基础设施搭建            | 🔴 最高 | 3-4        | 无       |
| **P1** | 核心纯逻辑单元测试          | 🔴 最高 | 8-10       | P0       |
| **P2** | Tushare 同步模块单元测试    | 🟠 高   | 8-10       | P0       |
| **P3** | 业务 Service 单元测试       | 🟠 高   | 12-15      | P0       |
| **P4** | Controller + Guard 集成测试 | 🟡 中   | 10-12      | P0       |
| **P5** | E2E API 流程测试            | 🟡 中   | 5-8        | P0, P4   |
| **P6** | WebSocket / Queue 集成测试  | 🔵 低   | 3-4        | P0       |

## 执行建议

### 分步落地顺序

```
P0 → P1（前 3 个文件）→ P2（前 4 个文件）→ P3（前 3 个文件）→ 回头补全 → P4 → P5 → P6
```

### 每个阶段的验收标准

| 阶段 | 验收标准                                                           |
| ---- | ------------------------------------------------------------------ |
| P0   | `pnpm test` 可运行，jest.config 无报错                             |
| P1   | 纯逻辑测试全绿，覆盖核心计算函数                                   |
| P2   | Tushare 同步链路测试全绿，覆盖所有 sync service 主流程             |
| P3   | 业务 service 测试全绿，auth / stock / backtest / factor 主流程覆盖 |
| P4   | Controller 测试全绿，DTO 校验 + Guard 拦截全覆盖                   |
| P5   | E2E 核心流程测试全绿                                               |
| P6   | 异步链路（Queue + WS）测试全绿                                     |

### Mock 策略总结

| 依赖        | Mock 方案                                     |
| ----------- | --------------------------------------------- |
| Prisma      | 手写 jest.fn() mock（参考现有 spec 风格）     |
| Redis       | 内存 Map 模拟 get/set/del                     |
| Tushare API | jest.fn() 返回固定 fixture 数据               |
| BullMQ      | mock Queue.add()、Job 内联处理                |
| Socket.IO   | mock EventsGateway 方法                       |
| bcrypt      | 真实调用（速度可接受）或 jest.mock 加速       |
| JWT         | 使用 `@nestjs/jwt` 测试 secret 生成真实 token |
