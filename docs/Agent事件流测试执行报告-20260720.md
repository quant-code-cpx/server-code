## 2. 真实 HTTP 验证

- 历史终态 Run 从 0 重放 18 个事件，sequence 为连续 1–18，最后事件 `agent.completed`。
- 使用第 17 个持久 eventId 作为 `Last-Event-ID`，Body 仍传 `afterSequence=0`，只重放 sequence 18，证明 Header 优先。
- 暂停 Worker 后发送新 Run，先建立 SSE；300 ms 后启动 Worker。同一连接持续 22.558 秒并收到完整 1–18，首事件 `message.created`、终态 `agent.completed`。
- 跨租户访问返回 HTTP 404/code 6002；无 JWT 返回 401。
- 响应为 `text/event-stream; charset=utf-8`，带 `no-cache, no-transform`、`X-Accel-Buffering: no`，不含 JSON wrapper。
- 实测指标：active connections 最终回到 0；replay 20、live 17、bytes 17044，三条连接均以 terminal 结束。

## 4. 测试覆盖

- serializer：14 个事件、UTF-8/CR/LF、防伪造 frame、未知类型/版本、非法 eventId。
- API：POST-only、严格 Body、401、404/6002、raw headers、heartbeat、Swagger 200。
- stream：high-water、Last-Event-ID 优先、replay/live 交界、gap fail-closed、终态、连接上限、shutdown。
- 背压：单帧/buffer 上限与 Writable 不 drain 超时，关闭原因为 `slow_consumer`。
- 数据：owner-scoped eventId、历史 INTERNAL sequence 修复、终态 trigger 下 migration、payload 回填。
- lifecycle：Transform bypass、HTTP logging/metrics close 结算、headersSent 后异常过滤器不再写 JSON。
