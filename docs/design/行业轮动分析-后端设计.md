## 一、功能总览

行业轮动分析模块帮助用户识别行业领涨/领跌格局、资金流动方向与估值高低，辅助行业配置决策。

| 模块                 | 接口路径                                    | 数据源表                                                         | 是否需新建表 | 状态    |
| -------------------- | ------------------------------------------- | ---------------------------------------------------------------- | ------------ | ------- |
| 行业收益对比         | `POST /industry-rotation/return-comparison` | `sector_capital_flows`                                           | 否           | 🆕 新建 |
| 行业动量排名         | `POST /industry-rotation/momentum-ranking`  | `sector_capital_flows`                                           | 否           | 🆕 新建 |
| 行业资金流转分析     | `POST /industry-rotation/flow-analysis`     | `sector_capital_flows`                                           | 否           | 🆕 新建 |
| 行业估值分位         | `POST /industry-rotation/valuation`         | `stock_daily_valuation_metrics` + `stock_basic_profiles`         | 否           | 🆕 新建 |
| 行业轮动总览         | `POST /industry-rotation/overview`          | 以上四张表的聚合                                                 | 否           | 🆕 新建 |
| 单行业详情           | `POST /industry-rotation/detail`            | `sector_capital_flows` + `stock_daily_valuation_metrics` + `stock_basic_profiles` | 否 | 🆕 新建 |
| 行业轮动热力图       | `POST /industry-rotation/heatmap`           | `sector_capital_flows`                                           | 否           | 🆕 新建 |

### 数据源评估

- **不需要新增 Tushare 数据同步**。所有功能均可基于已有数据表实现。
- 行业收益与资金流数据来源于 `sector_capital_flows`（Tushare `moneyflow_ind_dc`，`content_type = 'INDUSTRY'`），默认保留近 60 个交易日。
- 行业估值数据通过 `stock_daily_valuation_metrics`（`daily_basic`）按行业聚合计算，数据历史更长。
- 行业分类来源于 `stock_basic_profiles.industry` 字段（Tushare `stock_basic`）。

#### 可选增强数据（不影响基础实现）

| 接口名称           | 所需积分 | 说明                                                               |
| ------------------ | -------- | ------------------------------------------------------------------ |
| `index_classify`   | 120      | 申万行业分类，提供标准化的一级/二级行业索引                        |

| `sw_daily`         | 2000     | 申万行业指数日行情，可用于替代东财行业指数数据                      |

> 当前使用东财行业板块数据（`moneyflow_ind_dc`），可满足所有功能需求。如需更标准化的申万行业分类，后续可引入上述接口。

## 二、行业分类体系说明

本模块涉及两套行业分类：

| 分类来源                              | 数据表                   | 标识方式                 | 行业数量 | 用途                             |
| ------------------------------------- | ------------------------ | ------------------------ | -------- | -------------------------------- |
| 东财行业板块（`moneyflow_ind_dc`）    | `sector_capital_flows`   | 板块代码 `BK0475` + 名称 | ~80      | 行业收益、资金流、动量排名       |
| Tushare `stock_basic.industry`        | `stock_basic_profiles`   | 行业名称文本             | ~70      | 个股→行业映射、行业估值聚合      |

### 名称对齐策略

两套分类的行业名称大部分一致（如 "银行"、"电子"、"医药生物"），但可能存在少量差异。对齐规则：

1. **行业收益 / 资金流 / 动量**：直接使用 `sector_capital_flows` 的 `name` 字段，无需对齐。
2. **行业估值**：以 `stock_basic_profiles.industry` 为准，按行业名称聚合。
3. **总览 / 详情接口**中需要合并两套数据时，以 **行业名称精确匹配** 作为 JOIN 条件。对于无法匹配的行业，返回可用维度的数据，缺失维度标记为 `null`。

## 三、新建接口详细设计

### 3.1 `POST /industry-rotation/return-comparison` — 行业收益对比

**功能**：返回所有行业在多个时间窗口（5 / 20 / 60 日）的累计收益率，用于横向对比行业表现。

#### 请求 DTO：`ReturnComparisonQueryDto`

```typescript
class ReturnComparisonQueryDto {
  /** 查询截止日期（YYYYMMDD），默认最新交易日 */
  @IsOptional()
  @Matches(/^\d{8}$/)
  trade_date?: string

  /** 收益率计算窗口（天数），默认返回全部三个窗口 */
  @IsOptional()
  @IsArray()
  @IsInt({ each: true })
  @Min(1, { each: true })
  @Max(60, { each: true })
  periods?: number[] = [5, 20, 60]

  /** 排序依据的窗口天数，默认 20 */
  @IsOptional()
  @IsInt()
  sort_period?: number = 20

  /** 排序方向 */
  @IsOptional()
  @IsEnum(['asc', 'desc'])
  order?: 'asc' | 'desc' = 'desc'
}
```

#### 响应结构

```typescript
interface ReturnComparisonResponse {
  tradeDate: string
  /** 每个行业在各时间窗口的累计收益率 */
  industries: Array<{
    tsCode: string       // 东财板块代码（如 BK0475）
    name: string         // 行业名称
    /** 各窗口收益率，key 为天数，value 为累计收益率 % */
    returns: Record<number, number | null>
    /** 最新一日涨跌幅 % */
    latestPctChange: number | null
    /** 最新收盘价（板块指数） */
    latestClose: number | null
  }>
}
```

#### 实现要点

1. 从 `sector_capital_flows` 查询 `content_type = 'INDUSTRY'`，取目标日期往前 max(periods) 个交易日的数据。
2. 对每个行业（`ts_code`），基于 `close` 字段计算 N 日收益率：
   ```
   return_N = (close_latest / close_N_days_ago - 1) * 100
   ```
3. 使用 SQL 实现（避免大量应用层计算）：
   ```sql
   WITH ranked AS (
     SELECT
       ts_code,
       name,
       trade_date,
       close,
       pct_change,
       ROW_NUMBER() OVER (PARTITION BY ts_code ORDER BY trade_date DESC) AS rn
     FROM sector_capital_flows
     WHERE content_type = '行业'
       AND trade_date <= $targetDate
   )
   SELECT
     r0.ts_code,
     r0.name,
     r0.close AS latest_close,
     r0.pct_change AS latest_pct_change,
     r0.close / NULLIF(r5.close, 0) - 1 AS return_5,
     r0.close / NULLIF(r20.close, 0) - 1 AS return_20,
     r0.close / NULLIF(r60.close, 0) - 1 AS return_60
   FROM ranked r0
   LEFT JOIN ranked r5  ON r5.ts_code = r0.ts_code  AND r5.rn = 6    -- 5 日前
   LEFT JOIN ranked r20 ON r20.ts_code = r0.ts_code AND r20.rn = 21   -- 20 日前
   LEFT JOIN ranked r60 ON r60.ts_code = r0.ts_code AND r60.rn = 61   -- 60 日前
   WHERE r0.rn = 1
   ORDER BY return_20 DESC
   ```
   > **注意**：rn = N+1 表示 N 个交易日前的收盘价。若数据不足 N+1 天则该窗口返回 `null`。
4. 若 `periods` 包含自定义值，动态生成 LEFT JOIN（最多 5 个窗口以防滥用）。
5. 按 `sort_period` 对应的收益率列排序。

### 3.2 `POST /industry-rotation/momentum-ranking` — 行业动量排名

**功能**：基于加权动量评分对行业排名，综合短期（5 日）、中期（20 日）、长期（60 日）收益率，帮助识别具有持续趋势的行业。

#### 请求 DTO：`MomentumRankingQueryDto`

```typescript
class MomentumRankingQueryDto {
  /** 查询截止日期（YYYYMMDD），默认最新交易日 */
  @IsOptional()
  @Matches(/^\d{8}$/)
  trade_date?: string

  /** 动量评分方法 */
  @IsOptional()
  @IsEnum(['weighted', 'simple'])
  method?: 'weighted' | 'simple' = 'weighted'

  /**
   * 加权系数（仅 method='weighted' 时生效）
   * 格式 [短期权重, 中期权重, 长期权重]，默认 [0.3, 0.4, 0.3]
   */
  @IsOptional()
  @IsArray()
  weights?: number[] = [0.3, 0.4, 0.3]

  /** 返回 Top N，默认全部 */
  @IsOptional()
  @IsInt()
  @Min(1)
  @Max(100)
  limit?: number

  /** 排序方向 */
  @IsOptional()
  @IsEnum(['asc', 'desc'])
  order?: 'asc' | 'desc' = 'desc'
}
```

#### 响应结构

```typescript
interface MomentumRankingResponse {
  tradeDate: string
  method: 'weighted' | 'simple'
  industries: Array<{
    tsCode: string
    name: string
    /** 综合动量评分 */
    momentumScore: number
    /** 5 日收益率 % */
    return5d: number | null
    /** 20 日收益率 % */
    return20d: number | null
    /** 60 日收益率 % */
    return60d: number | null
    /** 最新一日涨跌幅 % */
    latestPctChange: number | null
    /** 排名（1 = 最强） */
    rank: number
  }>
}
```

#### 实现要点

1. 复用 3.1 的收益率计算逻辑，获取每个行业的 5 / 20 / 60 日收益率。
2. 计算动量评分：
   - **`weighted` 模式**：`score = w1 × return_5d + w2 × return_20d + w3 × return_60d`，权重归一化后计算。若某窗口数据不足，仅用可用窗口并重新归一化权重。
   - **`simple` 模式**：`score = return_20d`（中期收益即为动量分）。
3. 按 `momentumScore` 排序，添加排名字段。
4. 若指定 `limit`，截取 Top N。

### 3.3 `POST /industry-rotation/flow-analysis` — 行业资金流转分析

**功能**：分析行业间的资金流入流出格局，包括多日累计净流入、资金流与收益的关系、行业间资金转移信号。

#### 请求 DTO：`FlowAnalysisQueryDto`

```typescript
class FlowAnalysisQueryDto {
  /** 查询截止日期（YYYYMMDD），默认最新交易日 */
  @IsOptional()
  @Matches(/^\d{8}$/)
  trade_date?: string

  /** 分析周期（天数），默认 5 */
  @IsOptional()
  @IsInt()
  @Min(1)
  @Max(60)
  days?: number = 5

  /** 排序维度 */
  @IsOptional()
  @IsEnum(['cumulative_net', 'avg_daily_net', 'flow_momentum'])
  sort_by?: 'cumulative_net' | 'avg_daily_net' | 'flow_momentum' = 'cumulative_net'

  /** 排序方向 */
  @IsOptional()
  @IsEnum(['asc', 'desc'])
  order?: 'asc' | 'desc' = 'desc'

  /** Top N，默认全部 */
  @IsOptional()
  @IsInt()
  @Min(1)
  @Max(100)
  limit?: number
}
```

#### 响应结构

```typescript
interface FlowAnalysisResponse {
  tradeDate: string
  days: number
  industries: Array<{
    tsCode: string
    name: string
    /** 区间累计净流入（万元） */
    cumulativeNetAmount: number
    /** 日均净流入（万元） */
    avgDailyNetAmount: number
    /** 区间累计涨跌幅 % */
    cumulativeReturn: number | null
    /** 资金流动量：近 days/2 日净流入 - 前 days/2 日净流入（万元） */
    flowMomentum: number
    /** 资金流加速度：与前一个等长周期的净流入之差 */
    flowAcceleration: number | null
    /** 超大单累计净流入（万元） */
    cumulativeBuyElg: number
    /** 大单累计净流入（万元） */
    cumulativeBuyLg: number
    /** 主力占比：(超大单+大单) / 总净流入 */
    mainForceRatio: number | null
    /** 最新一日净流入排名 */
    latestDayRank: number | null
  }>
  /** 全市场汇总 */
  summary: {
    /** 净流入行业数 */
    inflowCount: number
    /** 净流出行业数 */
    outflowCount: number
    /** 净流入前 5 行业名称 */
    topInflowNames: string[]
    /** 净流出前 5 行业名称 */
    topOutflowNames: string[]
  }
}
```

#### 实现要点

1. 查询 `sector_capital_flows`，`content_type = 'INDUSTRY'`，取目标日期往前 `days × 2` 个交易日的数据（用于计算加速度）。
2. **累计净流入**：对近 `days` 天的 `net_amount` 求和。
3. **资金流动量**：将 `days` 天拆为前后两半，后半净流入 - 前半净流入。
4. **资金流加速度**：当前 `days` 天的累计净流入 - 前一个 `days` 天的累计净流入。
5. **主力占比**：`(sum(buy_elg_amount) + sum(buy_lg_amount)) / sum(net_amount)`。
6. SQL 实现：
   ```sql
   WITH date_range AS (
     -- 取近 days*2 个交易日，用于 acceleration 计算
     SELECT DISTINCT trade_date
     FROM sector_capital_flows
     WHERE content_type = '行业' AND trade_date <= $targetDate
     ORDER BY trade_date DESC
     LIMIT $days_x2
   ),
   recent AS (
     SELECT scf.*,
       ROW_NUMBER() OVER (PARTITION BY ts_code ORDER BY trade_date DESC) AS day_rn
     FROM sector_capital_flows scf
     JOIN date_range dr ON dr.trade_date = scf.trade_date
     WHERE scf.content_type = '行业'
   )
   SELECT
     ts_code,
     MAX(name) AS name,
     -- 近 days 天累计
     SUM(net_amount) FILTER (WHERE day_rn <= $days) AS cumulative_net,
     AVG(net_amount) FILTER (WHERE day_rn <= $days) AS avg_daily_net,
     -- 近 days 天累计超大单/大单
     SUM(buy_elg_amount) FILTER (WHERE day_rn <= $days) AS cum_buy_elg,
     SUM(buy_lg_amount) FILTER (WHERE day_rn <= $days) AS cum_buy_lg,
     -- 近半周期 vs 前半周期（资金流动量）
     SUM(net_amount) FILTER (WHERE day_rn <= $half_days) AS recent_half_net,
     SUM(net_amount) FILTER (WHERE day_rn > $half_days AND day_rn <= $days) AS earlier_half_net,
     -- 前一个周期（加速度）
     SUM(net_amount) FILTER (WHERE day_rn > $days AND day_rn <= $days_x2) AS prev_period_net,
     -- 收益率（从 close 计算）
     -- ...
   FROM recent
   GROUP BY ts_code
   ```
7. 在 service 层计算 `flowMomentum = recent_half_net - earlier_half_net` 和 `flowAcceleration = cumulative_net - prev_period_net`。
8. `summary` 在 service 层聚合：统计 `cumulativeNetAmount > 0` 的数量即为 `inflowCount`。

### 3.4 `POST /industry-rotation/valuation` — 行业估值分位

**功能**：计算每个行业的 PE_TTM / PB 中位数，并给出在 1 年 / 3 年历史中的百分位排名，帮助判断行业估值高低。

#### 请求 DTO：`IndustryValuationQueryDto`

```typescript
class IndustryValuationQueryDto {
  /** 查询日期（YYYYMMDD），默认最新交易日 */
  @IsOptional()
  @Matches(/^\d{8}$/)
  trade_date?: string

  /** 筛选特定行业（行业名称），不传则返回全部 */
  @IsOptional()
  @IsString()
  industry?: string

  /** 排序字段 */
  @IsOptional()
  @IsEnum(['pe_ttm', 'pb', 'pe_percentile_1y', 'pb_percentile_1y'])
  sort_by?: 'pe_ttm' | 'pb' | 'pe_percentile_1y' | 'pb_percentile_1y' = 'pe_percentile_1y'

  /** 排序方向 */
  @IsOptional()
  @IsEnum(['asc', 'desc'])
  order?: 'asc' | 'desc' = 'asc'
}
```

#### 响应结构

```typescript
interface IndustryValuationResponse {
  tradeDate: string
  industries: Array<{
    /** 行业名称（来自 stock_basic_profiles.industry） */
    industry: string
    /** 行业内上市公司数量 */
    stockCount: number
    /** 当日行业 PE_TTM 中位数 */
    peTtmMedian: number | null
    /** 当日行业 PB 中位数 */
    pbMedian: number | null
    /** PE_TTM 在 1 年历史中的百分位（0-100） */
    peTtmPercentile1y: number | null
    /** PE_TTM 在 3 年历史中的百分位（0-100） */
    peTtmPercentile3y: number | null
    /** PB 在 1 年历史中的百分位（0-100） */
    pbPercentile1y: number | null
    /** PB 在 3 年历史中的百分位（0-100） */
    pbPercentile3y: number | null
    /** 估值状态标签 */
    valuationLabel: '低估' | '适中' | '偏高' | '高估'
  }>
}
```

#### 实现要点

##### 步骤 1：计算当日各行业 PE/PB 中位数

```sql
SELECT
  sb.industry,
  COUNT(*)::int AS stock_count,
  PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY db.pe_ttm) AS pe_ttm_median,
  PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY db.pb)     AS pb_median
FROM stock_daily_valuation_metrics db
JOIN stock_basic_profiles sb ON sb.ts_code = db.ts_code
WHERE db.trade_date = $targetDate
  AND sb.list_status = 'L'
  AND sb.industry IS NOT NULL AND sb.industry != ''
  AND db.pe_ttm > 0 AND db.pe_ttm < 1000
  AND db.pb > 0
GROUP BY sb.industry
```

##### 步骤 2：计算历史分位（以 1 年 PE_TTM 为例）

```sql
-- 取过去 1 年每个交易日的行业 PE 中位数
WITH daily_medians AS (
  SELECT
    sb.industry,
    db.trade_date,
    PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY db.pe_ttm) AS pe_ttm_median
  FROM stock_daily_valuation_metrics db
  JOIN stock_basic_profiles sb ON sb.ts_code = db.ts_code
  WHERE db.trade_date >= $oneYearAgo AND db.trade_date <= $targetDate
    AND sb.list_status = 'L'
    AND sb.industry IS NOT NULL AND sb.industry != ''
    AND db.pe_ttm > 0 AND db.pe_ttm < 1000
    AND db.pb > 0
  GROUP BY sb.industry, db.trade_date
)
SELECT
  industry,
  -- 当日中位数在历史中的百分位排名
  PERCENT_RANK() OVER (
    PARTITION BY industry
    ORDER BY pe_ttm_median
  ) * 100 AS percentile
FROM daily_medians
WHERE trade_date = $targetDate
```

> **性能警告**：步骤 2 涉及对 `stock_daily_valuation_metrics` 全表的大范围 JOIN + GROUP BY + 窗口函数，计算量极大。优化策略：
>
> 1. **Redis 缓存**：TTL 设为 8 小时（盘后数据不变），Cache Key 包含 `trade_date`。
> 2. **分行业并行计算**：可将步骤 2 按行业拆分为多个查询，利用 `Promise.all` 并行执行。
> 3. **降采样**：对 3 年窗口可按周采样（取每周最后一个交易日），减少计算量。
> 4. **预计算表（v2 考虑）**：若性能仍不达标，可新增 `IndustryValuationSnapshot` Prisma Model，通过定时任务每日预计算落库。

##### 步骤 3：估值标签

在 service 层根据 1 年 PE 百分位生成标签：

| 百分位区间   | 标签   |
| ------------ | ------ |
| 0 ~ 25       | 低估   |
| 25 ~ 50      | 适中   |
| 50 ~ 75      | 偏高   |
| 75 ~ 100     | 高估   |

### 3.5 `POST /industry-rotation/overview` — 行业轮动总览

**功能**：一次性返回行业轮动四个维度（收益、动量、资金、估值）的核心摘要数据，作为仪表盘首屏接口。

#### 请求 DTO：`RotationOverviewQueryDto`

```typescript
class RotationOverviewQueryDto {
  /** 查询日期（YYYYMMDD），默认最新交易日 */
  @IsOptional()
  @Matches(/^\d{8}$/)
  trade_date?: string
}
```

#### 响应结构

```typescript
interface RotationOverviewResponse {
  tradeDate: string
  /** 收益维度：涨幅前 5 / 跌幅前 5 行业 */
  returnSnapshot: {
    topGainers: Array<{ name: string; return20d: number }>
    topLosers: Array<{ name: string; return20d: number }>
  }
  /** 动量维度：动量评分前 5 / 后 5 行业 */
  momentumSnapshot: {
    leaders: Array<{ name: string; momentumScore: number }>
    laggards: Array<{ name: string; momentumScore: number }>
  }
  /** 资金维度：5 日净流入前 5 / 净流出前 5 行业 */
  flowSnapshot: {
    topInflow: Array<{ name: string; cumulativeNetAmount: number }>
    topOutflow: Array<{ name: string; cumulativeNetAmount: number }>
  }
  /** 估值维度：最低估前 5 / 最高估前 5 行业 */
  valuationSnapshot: {
    undervalued: Array<{ name: string; peTtmPercentile1y: number }>
    overvalued: Array<{ name: string; peTtmPercentile1y: number }>
  }
}
```

#### 实现要点

1. 内部并行调用前 4 个接口的 service 方法（不通过 HTTP，直接调用 service 层）：
   ```typescript
   const [returns, momentum, flow, valuation] = await Promise.all([
     this.getReturnComparison({ trade_date, periods: [20], sort_period: 20, order: 'desc' }),
     this.getMomentumRanking({ trade_date, method: 'weighted', order: 'desc' }),
     this.getFlowAnalysis({ trade_date, days: 5, sort_by: 'cumulative_net', order: 'desc' }),
     this.getIndustryValuation({ trade_date, sort_by: 'pe_percentile_1y', order: 'asc' }),
   ])
   ```
2. 从各维度中分别取前 5 和后 5 组装响应。
3. 本接口无独立缓存，依赖子接口各自的缓存。

### 3.6 `POST /industry-rotation/detail` — 单行业详情

**功能**：返回指定行业的收益趋势、资金流趋势、估值趋势以及成分股信息，用于行业钻取页面。

#### 请求 DTO：`IndustryDetailQueryDto`

```typescript
class IndustryDetailQueryDto {
  /** 行业名称（如 "银行"、"电子"） */
  @IsString()
  @IsNotEmpty()
  industry: string

  /** 趋势数据天数，默认 20 */
  @IsOptional()
  @IsInt()
  @Min(5)
  @Max(60)
  days?: number = 20
}
```

#### 响应结构

```typescript
interface IndustryDetailResponse {
  industry: string
  /** 东财板块代码（可能为 null，若名称未匹配到 sector_capital_flows） */
  tsCode: string | null

  /** 收益走势（日序列） */
  returnTrend: Array<{
    tradeDate: string
    close: number        // 板块收盘指数
    pctChange: number    // 当日涨跌幅 %
    cumulativeReturn: number // 从序列起点开始的累计收益率 %
  }>

  /** 资金流趋势（日序列） */
  flowTrend: Array<{
    tradeDate: string
    netAmount: number       // 当日净流入（万元）
    cumulativeNet: number   // 累计净流入
    buyElgAmount: number    // 超大单
    buyLgAmount: number     // 大单
  }>

  /** 估值快照 */
  valuation: {
    peTtmMedian: number | null
    pbMedian: number | null
    peTtmPercentile1y: number | null
    pbPercentile1y: number | null
    valuationLabel: string
  } | null

  /** 成分股概览（按市值排序，Top 20） */
  topStocks: Array<{
    tsCode: string
    name: string
    pctChg: number | null     // 最新日涨跌幅
    peTtm: number | null
    pb: number | null
    totalMv: number | null    // 总市值（万元）
  }>
}
```

#### 实现要点

1. **行业名称 → 东财板块代码映射**：
   ```sql
   SELECT DISTINCT ts_code, name
   FROM sector_capital_flows
   WHERE content_type = '行业' AND name = $industryName
   LIMIT 1
   ```
   若无匹配结果，`tsCode` 和 `returnTrend` / `flowTrend` 为空。

2. **收益走势**：查 `sector_capital_flows` 近 `days` 天，计算 `cumulativeReturn`：
   ```typescript
   cumulativeReturn = (close / firstClose - 1) * 100
   ```

3. **资金流趋势**：复用 Market 模块 `getSectorFlowTrend` 类似逻辑。

4. **估值快照**：调用 3.4 接口的 service 方法，传入 `industry` 参数，取单行业结果。

5. **成分股概览**：
   ```sql
   SELECT
     sb.ts_code, sb.name,
     d.pct_chg,
     db.pe_ttm, db.pb, db.total_mv
   FROM stock_basic_profiles sb
   LEFT JOIN stock_daily_prices d ON d.ts_code = sb.ts_code AND d.trade_date = $latestTradeDate
   LEFT JOIN stock_daily_valuation_metrics db ON db.ts_code = sb.ts_code AND db.trade_date = $latestTradeDate
   WHERE sb.industry = $industryName AND sb.list_status = 'L'
   ORDER BY db.total_mv DESC NULLS LAST
   LIMIT 20
   ```

## 七、性能与扩展考量

### 7.1 行业估值分位的性能优化路径

行业估值分位（3.4）是本模块计算最密集的接口。优化分三阶段：

**Phase 1（初始实现）**：
- 使用 Raw SQL + Redis 缓存（8h TTL）
- 限制历史窗口为 1 年和 3 年（不提供 5 年窗口）
- 预计单次请求耗时：3-10 秒（取决于数据量）

**Phase 2（若 Phase 1 不达标）**：
- 降采样：3 年窗口按周采样（每周最后一个交易日的中位数）
- 并行化：按行业拆分为独立查询，使用 `Promise.all` 并行

**Phase 3（长期优化）**：
- 新增 `IndustryValuationSnapshot` Prisma Model：
  ```prisma
  model IndustryValuationSnapshot {
    industry     String   @db.VarChar(64)
    tradeDate    DateTime @map("trade_date") @db.Date
    stockCount   Int      @map("stock_count")
    peTtmMedian  Float?   @map("pe_ttm_median")
    pbMedian     Float?   @map("pb_median")
    syncedAt     DateTime @default(now()) @map("synced_at")

    @@id([industry, tradeDate])
    @@index([tradeDate])
    @@map("industry_valuation_snapshots")
  }
  ```
- 定时任务每日盘后预计算，估值分位查询直接读预计算表。

### 7.2 与现有模块的关系

| 现有接口                                 | 本模块接口                            | 关系说明                                                   |
| ---------------------------------------- | ------------------------------------- | ---------------------------------------------------------- |
| `POST /market/sector-ranking`            | `return-comparison`                   | sector-ranking 仅返回当日涨跌幅排行；本模块增加多周期收益  |
| `POST /market/sector-flow-ranking`       | `flow-analysis`                       | sector-flow-ranking 为单日快照；本模块增加多日累计与动量    |
| `POST /market/sector-flow-trend`         | `detail` 中的 flowTrend               | 功能类似，本模块额外提供估值和成分股                       |
| `POST /market/valuation`                 | `valuation`                           | market/valuation 为全 A 市场；本模块按行业细分              |

> **不修改现有接口**。本模块作为独立功能模块，与 Market 模块并行存在。共享同一底层数据表。

### 7.3 后续扩展方向

1. **概念板块轮动**：复用同一模块架构，将 `content_type` 从 `INDUSTRY` 扩展到 `CONCEPT`。
2. **行业轮动信号**：基于动量排名变化生成买入/卖出信号。
3. **行业相关性矩阵**：计算行业间收益率相关系数矩阵。
4. **行业轮动回测**：将行业动量策略接入回测框架。
